## Phase 1：架构整理

### 目标
代码从 partial class 升级为 MVVM。**功能行为与 v1.0.0 完全一致——用户看到的界面一模一样。**

### 实施顺序（先验证，再搬家）

```
1. 读 v1 代码 → 理解 6 个 partial class 的职责和依赖关系
2. NuGet 引入 CommunityToolkit.Mvvm
3. DataConverter.cs（纯静态、无依赖 → 马上可验证）
4. ProtocolParser.cs（无外部依赖，输入字符串输出 ParseResult）
5. SerialPortSession.cs（从 MainWindow 搬家，依赖 System.IO.Ports）
6. MainViewModel.cs（骨架，持有 SerialPortSession）
7. PlotViewModel / SliderPanelViewModel / KeyPanelViewModel / DisplayPanelViewModel
   （全骨架——空类，只放一个 Title 属性，为后续 Phase 占位）
8. PreferenceService.cs + prefs.json（替代 window.cfg）
9. MainWindow.xaml.cs 瘦身 + MainWindow.xaml 调整绑定
   （删掉搬走的方法，改为调用 Session/VM，不改 UI 布局）
10. 编译验证 → 行为与 v1.0.0 完全一致
```

**原则**：先建无依赖的（DataConverter、ProtocolParser），再建依赖少的（VM 骨架），最后动 MainWindow。每一步编译通过再走下一步。

### 1.1 提取 SerialPortSession

```
新文件：SerialPortSession.cs

从 MainWindow.xaml.cs 搬出：
  - serialPort / isSerialOpen
  - byteBuffer / receiveLineBuffer / flushTimer
  - OpenSerialPort() / CloseSerialPort()
  - serialPort_DataReceived()（字节读取 + Dispatcher.Invoke）
  - SendData() / SendRaw()
  - txByteCount / rxByteCount
  - DTR/RTS 控制
```

类签名：
```csharp
public class SerialPortSession : IDisposable
{
    public event Action<string> LineReceived;   // 完整行文本
    public event Action<bool> ConnectionChanged; // 连接状态变化
    public bool IsOpen { get; }
    public long TxBytes { get; }
    public long RxBytes { get; }
    public string PortName { get; }

    public void Open(string portName, int baudRate, ...);
    public void Close();
    public void Send(string content);
    public void SetDtr(bool enable);
    public void SetRts(bool enable);
}
```

**改动原则**：只搬家，不改逻辑。MainWindow 持有 `SerialPortSession _session`。

### 1.2 提取 DataConverter

```
新文件：DataConverter.cs

public static class DataConverter
{
    public static string BytesToHex(byte[] bytes);        // 逻辑不动
    public static byte[] HexToBytes(string str);           // 逻辑不动
    public static string BytesToText(byte[], string enc);  // 逻辑不动
```

### 1.3 提取 ProtocolParser

```
新文件：ProtocolParser.cs

public class ProtocolParser
{
    public static ParseResult Parse(string rawLine);
}

public class ParseResult
{
    public List<ProtocolMessage> Messages;  // [type, arg1, arg2, ...]
    public string PlainText;                // 方括号外的普通文本
}
```

解析规则：
- 匹配 `[...]`，支持引号包裹参数 `"..."`
- `[plot,P,0.5]` → `{Type:"plot", Args:["P","0.5"]}`
- `[display,0,0,"hello,18",24]` → `{Type:"display", Args:["0","0","hello,18","24"]}`
- 括号外的文本归入 `PlainText`

### 侧面板折叠逻辑

`BtnPanelCollapse_Click` 在 C# 中实现：

### 标签切换逻辑（C#，几乎不改）

代码与顶部标签栏版本完全一致，复用：
- `_currentTab`：当前选中标签名
- `_previousContentTab`：上一个内容标签（设置图标选中时，主区保持此标签内容不变）
- `TabContent_Checked`：内容图标点击 → 更新 `_currentTab` 和 `_previousContentTab` → `RefreshContentVisibility()`
- `TabSettings_Checked`：设置图标点击 → 仅更新 `_currentTab`（不更新 `_previousContentTab`）→ `RefreshContentVisibility()`
- `RefreshContentVisibility()`：主内容区用 `_previousContentTab` 决定，侧面板用 `_currentTab` 决定

### 3.2 实现

**PlotViewModel**：
```csharp
public class PlotViewModel : ObservableObject
{
    public PlotModel Model { get; } = new PlotModel();
    private Dictionary<string, LineSeries> _series = new();
    private const int MaxPoints = 5000;

    public void OnPlotMessage(string name, double value, DateTime timestamp)
    {
        if (!_series.TryGetValue(name, out var series))
        {
            series = new LineSeries { Title = name };
            Model.Series.Add(series);
            _series[name] = series;
        }
        series.Points.Add(new DataPoint(DateTimeAxis.ToDouble(timestamp), value));
        if (series.Points.Count > MaxPoints) series.Points.RemoveAt(0);
        Model.InvalidatePlot(true);

        // 更新数值面板
        UpdateValueDisplay(name, value);
    }
}
```

