# Serial Monitor V2 — 评估标准

> **用途**：供新 AI 对话独立对 V2 工程进行多维度打分。
> **使用方式**：先读 `当前状态.md` 建立全局认知，再按本文档逐维审查代码并打分。

---

## 评估方法论

### 原则
1. **基于代码审查，不依赖实际运行**——评审者无需 STM32 硬件和串口环境
2. **以用户视角为原点**——用户是新手小白，双击 exe 即用；不与 VS2022/命令行/Web 工具对比
3. **承认约束的价值**——WPF 固有限制（142MB 自包含、无裁剪）不应扣分
4. **区分"代码问题"和"架构局限"**——如 WPF 单文件体积是框架固有问题，不应扣代码分

### 评审流程
1. 读 `当前状态.md` 前两节（项目概览 + 硬性约束）
2. 通读 `Core/Services/`、`Core/ViewModels/`、`MainWindow.*.cs`、`MainWindow.xaml`
3. 按下方 6 个维度逐项打分
4. 每个维度给出 **得分** + **3 条以内的关键证据**（正面和负面）
5. 加权求和得出总分

## 评分维度

| # | 维度 | 权重 | 考察什么 |
|:--:|------|:---:|------|
| 1 | 功能完整性 | 25% | 串口调试工具 + 嵌入式交互工作站的功能覆盖率 |
| 2 | 代码架构 | 20% | MVVM 分层、职责分离、可读性、测试覆盖 |
| 3 | UI/UX 设计 | 20% | VS Code 风格布局、设计系统一致性、动效、主题 |
| 4 | 健壮性 | 15% | 异常处理、资源上限、边界情况、崩溃可追踪 |
| 5 | 工程实践 | 10% | Git 纪律、文档质量、构建流程、测试 |
| 6 | 性能优化 | 10% | 渲染效率、内存管理、单文件部署 |

## 一、功能完整性（权重 25%）

### 评分锚点

| 分数 | 标准 |
|:---:|------|
| 95-100 | 波形+控制面板+搜索+绘图+多串口+脚本——嵌入式交互工作站 |
| 85-94 | 波形+控制面板+所有常用功能+搜索+协议系统 |
| 75-84 | 基本收发+常用功能+部分高级功能 |
| 60-74 | 基本收发可用，功能有缺失 |
| <60 | 无法完成基本串口通信 |

### 审查清单

检查以下功能是否 **存在且代码路径完整**：

| 功能 | 审查位置 |
|------|---------|
| 串口打开/关闭（含 5 层异常） | `MainWindow.xaml.cs` → `OpenSerialPort()` |
| HEX/文本收发 + GBK/UTF-8 | `MainWindow.xaml.cs` → `cbReceiveMode` / `cbSendMode` |
| 波特率预设 + 自定义 | `MainWindow.xaml` → `cbBaudRate` (`IsEditable="True"`) |
| 硬件流控 / DTR / RTS | `MainWindow.Theme.cs` → `ApplyTheme` |
| 时间戳（3 格式） | `MainWindow.Log.cs` → `LogReceived` |
| 定时发送 + 换行符可选 | `MainWindow.xaml.cs` |
| 发送后自动清空 | `MainWindow.xaml.cs` → `SendData` |
| USB 热插拔 + 自动重连 | `MainWindow.xaml.cs` → `WndProc` |
| 流量统计 + 持久化 | `MainWindow.xaml.cs` |
| 快捷发送按钮面板 | `MainWindow.QuickSend.cs` |
| 日志导出 | `MainWindow.xaml.cs` → `btnExportLog_Click` |
| 行号 + 暂停/恢复 | `MainWindow.xaml` / `MainWindow.Log.cs` |
| 暗色/亮色主题 | `MainWindow.Theme.cs` → `ApplyTheme` |
| 搜索（Ctrl+F） | AvalonEdit `SearchPanel.Install` |
| 崩溃日志 | `App.xaml.cs` |
| **波形面板（OxyPlot）** | `PlotViewModel.cs` + `MainWindow.xaml` `panelPlot` |
| **HUD + 滚动/扫描双模式** | `PlotViewModel.cs` |
| **4 控制面板（按键/滑杆/摇杆/OLED）** | `MainWindow.Keys.cs` / `.Sliders.cs` / `.Joystick.cs` / `.Display.cs` |
| **协议系统 `[type,arg,...]`** | `Core/Services/ProtocolParser.cs` |
| **快捷键提示 + 使用示例页** | `MainWindow.xaml.cs` → `PopulateShortcutPage()` / `PopulateExamplesPage()` |
| **关于页（运行时版本+架构+Issue）** | `MainWindow.xaml` / `.xaml.cs` |

### 扣分指南
- 无波形交互（缩放/拖拽/游标）：-3
- 无多串口：-2
- 无脚本/宏：-1
- OLED 只能显示文本不能画图：-1

## 二、代码架构（权重 20%）

### 评分锚点

| 分数 | 标准 |
|:---:|------|
| 95-100 | 完整 MVVM、DI、接口抽象、测试覆盖 >80% |
| 85-94 | 清晰分层、Core 独立可测、合理 partial 拆分、有测试 |
| 75-84 | 基本拆分、主要路径有注释、少量重复 |
| 60-74 | 可工作但大文件堆积、方法过长 |
| <60 | 无法维护的意大利面代码 |

### 审查清单

| 检查项 | 审查位置 |
|--------|---------|
| Core/Services 层独立度 | 4 文件（DataConverter/ProtocolParser/SerialPortSession/PreferenceService）|
| Core/ViewModels 层 | 6 个 VM 文件 |
| partial class 拆分数 | 10 个 `MainWindow.*.cs` |
| 主文件行数 | `MainWindow.xaml.cs` |
| 单元测试 | `SerialMonitor.Tests/ProtocolParserTests.cs` |
| DataConverter 是否可测 | 4 个纯静态方法 |
| 协议解析是否可测 | `ProtocolParser.Parse()` 纯逻辑 |
| 命名一致性 | 中文命名空间 + PascalCase 类名 |

### 扣分指南
- 主文件 >2000 行：-3 ~ -5
- ViewModels 贫血（仅数据容器）：-2
- 无接口/DI：-2
- DataConverter 无测试：-5
- 控制面板主题切换全量重建 UI：-3

## 四、健壮性（权重 15%）

### 评分锚点

| 分数 | 标准 |
|:---:|------|
| 95-100 | 完善的异常处理链、自动恢复、数据完整性保证、全面边界测试 |
| 85-94 | 多层异常分类、资源上限保护、关键路径有 try-catch、崩溃可追踪 |
| 75-84 | 基本异常处理、部分资源上限、有日志 |
| 60-74 | 能跑但有未处理异常导致崩溃的风险 |
| <60 | 常见操作崩溃 |

### 审查清单

| 检查项 | 审查位置 |
|--------|---------|
| 串口打开 5 层异常 + 系统日志 | `OpenSerialPort()` |
| 后台线程 Read() try-catch | `SerialPortSession.OnDataReceived()` |
| 发送 Write() try-catch | `SerialPortSession.SendBytes()` |
| 协议路由 try-catch | `MainWindow.OnLineReceived()` |
| SendRaw try-catch | `MainWindow.SendRaw()` |
| 日志行数上限 | `MaxLogLines = 2000` |
| 暂停缓冲上限 | `MaxPausedLines = 2000` |
| 崩溃日志双通道 | `App.xaml.cs` |
| 死锁防护 | DataReceived 退订在 Close 之前 |
| 自动重连超时 | 500ms × 10 次 |

### 扣分指南
- 暂停缓冲满时静默丢数据：-2
- 协议消息无合法性校验（如 key 的 state 不校验 "down"/"up"）：-2
- HexToBytes 静默剥离非法字符，用户无感知：-3
- 无 CRC/数据校验：-1
