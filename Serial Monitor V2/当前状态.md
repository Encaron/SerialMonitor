## 一、项目起源

### 1.1 一切的开始：改了三个默认值

用户使用 B 站 UP 主**江科大**的 WinForms 串口助手 V1.1（位于 `../串口助手/`），功能完善但默认参数对不上：

| 设置 | 江科大默认 | 用户习惯 |
|------|:---:|:---:|
| 波特率 | 9600 | **115200** |
| 收发模式 | HEX | **文本** |
| 编码 | GBK | **UTF-8** |

每次打开软件都要改这三个选项——于是有了第一个 commit：改默认值。**整个项目的精神基因就是"工具应该记住我怎么用"。**

### 1.2 为什么开 WPF 版

WinForms 功能够了但界面太丑——无圆角、无动画、无阴影、无暗色主题。WPF 是最佳折中：串口逻辑复用江科大 `System.IO.Ports`（成熟稳定），UI 完全重写。

53 个 commit、四轮完善后，v1.0.0 得分 84.5/100，30+ 功能稳定运行。

### 1.3 为什么开 V2

v1.0.0 是**纯数据收发工具**。用户想要的是**理解数据含义的工具**：

```
STM32 发送：  Serial_Printf(&huart1, "[plot,P,%f][plot,I,%f][slider,kp,%f]\r\n", p, i, kp);

PC 软件收到：
  接收区：    14:23:05.123 -> [plot,P,0.5][plot,I,0.3][slider,kp,17]    ← 原样显示
  波形图：    两条曲线 P / I 实时更新                                      ← 新增
  滑杆面板：  kp = 17%，可拖拽发回 STM32                                  ← 新增
```

同时借这个机会把代码架构从 partial class 升级为 MVVM，为多串口等扩展留空间。

## 三、V2 新增内容概览

### 3.1 协议格式（规范化）

```
格式：  [类型, 参数1, 参数2, ...]
       ↑ 方括号包裹，一条消息一个括号。多括号可同行，普通文本可混排。

类型：
  plot           [plot,曲线名,数值]             波形图数据，曲线名自动成为图例
  plot-clear     [plot-clear]                   清空波形图
  slider         [slider,名称,百分比]            滑杆控件
  key            [key,名字,状态]                按键控件（状态: down/up）
  joystick       [joystick,编号,h1,v1,h2,v2]    摇杆控件
  display        [display,x,y,"文本",字号]      虚拟 OLED，文本含逗号须用引号包裹
  display-clear  [display-clear]                清空虚拟 OLED

示例：
  [key,MyDownKey,down]     按键名字不限于编号，任意字符串
  [display,0,0,"hello,18",24]   引号内逗号不计为分隔符
  [display,0,0,hello,18]        无逗号时引号可省
```

STM32 用法：
```c
// PID 调参
Serial_Printf(&huart1, "[plot,P,%f][plot,I,%f][plot,D,%f]\r\n", p, i, d);
Serial_Printf(&huart1, "[slider,kp,%f]\r\n", kp_slider_value);

// 加速度计
Serial_Printf(&huart1, "[plot,ax,%f][plot,ay,%f][plot,az,%f]\r\n", ax, ay, az);
// 三条曲线自动创建，名字自动成为图例，颜色自动分配
```

### 3.2 数据流

```
单向（STM32 → PC）：
  串口字节 → DataReceived → 文本行 →
    ├── ProtocolParser
    │   ├── [plot,...]  → PlotViewModel       （追加数据点）
    │   ├── [slider,...]→ SliderPanelViewModel（更新滑杆位置）
    │   ├── [key,...]   → KeyPanelViewModel   （更新按键状态）
    │   ├── [display,...]→ DisplayPanelViewModel（渲染到虚拟 OLED）
    │   └── 普通文本    → 接收区（原样显示）    ← 行为完全不变
    │
    └── 接收区：LogReceived(text) ← 跟现在一模一样

双向（PC → STM32）：
  控件面板操作 → 编码为 [type,...] → SerialPortSession.Send() → 串口发出
    ├── 拖滑杆 kp → "[slider,kp,17]\r\n"
    ├── 点按键 4  → "[key,4,down]\r\n"
    └── STM32 通过 @TEXT 协议接收 → 拆包执行
```

### 3.4.4 Phase 3 踩坑记录（⚠️ 新 AI 必读）

| # | 坑 | 现象 | 根因 | 解决方案 |
|:--:|-----|------|------|------|
| **B1** | `oxy:PlotView` 放 XAML | 启动即崩溃，`TargetInvocationException` | 旧式 csproj 的 BAML 编译器加载第三方 WPF 程序集时失败 | **OxyPlot 控件全部在 C# 中 `new`，不走 XAML**。XAML 只保留 `xmlns:oxy` 命名空间声明 |

| **B4** | `_yAxis.Minimum/Maximum` 设置无效 | "适应"按钮不工作 | OxyPlot 2.1.0 需用 `Axis.Zoom(min, max)` 而非直接设 Minimum/Maximum | 统一用 `_yAxis.Zoom(min, max)` |
| **B5** | `RecalcYAxis` 只扫描 `_series` 字典 | 测试曲线（直接加在 `Model.Series` 里）被忽略 | `_series` 只包含通过 `OnPlotMessage` 创建的曲线 | 扫描时遍历 `Model.Series`（所有曲线），不依赖 `_series` |
| **B6** | 每个数据点都 `InvalidatePlot(true)` | 数据频率高时界面卡顿 | 全量重绘开销大 | 30Hz 限流：`_dirty` 标记 + 33ms 最小间隔 |

> **⚠️ 2026-06-12 修订**：以下决策中的"右侧面板"在最终实现中已移至**左侧第二列**（VS Code Activity Bar 风格），但逻辑行为完全不变。阅读时将"右侧面板"理解为"侧面板"即可。

## 七、给新 AI 的补充说明

> 以下内容不在决策表里，是前面几轮 AI 在讨论中积累的实操经验。新 AI 读完后能少踩坑。

### v1 代码里藏着的坑

- **partial class 跨文件依赖**：`MainWindow.xaml.cs`（1370 行）里定义的字段被 `.Log.cs`、`.QuickSend.cs`、`.Settings.cs` 引用。搬家时注意依赖——比如 `_lineRoles: List<string>` 在 `.cs` 里定义但在 `.Log.cs` 里用。提取到新类时先 grep 确认所有引用点
- **AvalonEdit 不走 MVVM**：它的 `TextArea` 操作是 WinForms 风格的直接操作，Phase 1 不要试图把它改成绑定，保持 code-behind 方式。Phase 2 搬到接收区标签时照搬现有代码即可
- **DataReceived 线程安全**：串口事件在后台线程触发，所有 UI 操作必须 `Dispatcher.Invoke`。v1 已经做了，搬家时别丢
- **等宽字体回退链**：`Sarasa Mono SC` → `Consolas` → `Courier New`，不可删
- **行缓冲 100ms**：空闲 100ms 后强制输出残留文本，搬家到 `SerialPortSession` 时原样保留
- **智能滚屏 ≤8px**：ScrollViewer 距底部 ≤8px 时自动滚屏，搬家到接收区标签时保留
- **ComboBox 模板**：v1 的 ComboBox ControlTemplate 已含 `IsEditable` Trigger，新增 ComboBox 不要重新发明——复制现有模板改

### 实施时的优先级直觉

- **先做无依赖的**：DataConverter（纯静态）→ ProtocolParser（纯逻辑）→ SerialPortSession（依赖串口）→ ViewModel 骨架。每步编译通过再走下一步
- **Phase 2 的第 1 步（重写 MainWindow.xaml 壳子）是整个项目最重要的单步**。壳子跑通了（标签能切、面板能切、串口能开），后面全是填充
- **接收区迁移是 Phase 2 工作量最大的单步**——8 个功能（行号/系统消息/时间戳/回显/暂停/搜索/滚屏/导出）要逐条对照 2.8 检查清单。做完后发一版 exe 给用户确认
- **Phase 3 先验证 OxyPlot 能渲染一根静态线**，再堆 UI。别先建完整的 UI 再试——如果 OxyPlot 有问题（概率极低），早发现

### 几个文档里没写但讨论中强调过的点

- **键盘布局的 ⇧ 键是 ToggleButton（UI 控件），不是协议按键**——它只控制按键创建时的大小写模式，本身不发送任何数据。Enter/Backspace/Tab 等则可以映射为协议按键
- **键盘布局是追加模式**，不会覆盖已手动创建的按键。用户可以混搭手动键 + 键盘布局键
- **暂停按钮文字和图标都切换**（⏸暂停 ↔ ▶继续），固定宽度避免工具栏跳动
- **方向键预设要做十字排列**（↑在上 ↓在下 ←左 →右），不是横排四个键
- **数值 HUD 不是窗口、不是弹窗、不可拖拽**——它是 Grid 同一格里的 Border，半透明底，鼠标穿透。新 AI 容易习惯性地做一个独立窗口，不要

### 什么不要做

- 不要做 Canvas 自由定位（决策 28）
- 不要做面板拖拽调整宽度（决策 27）——固定 220px
- 不要改 4 个数据转换方法的算法（约束 1）
- 不要用 `{StaticResource}`（约束 7）——全 XAML 96 处 DynamicResource，新增必须对齐
- 不要给绘图加独立的"数值方框"切分波形区（决策 21）——数值是图上 HUD
- 不要把暂停/清除/CSV 放进右侧面板——它们在顶部工具栏（决策 20）
- 不要碰 `../串口助手-WPF/` 目录——那是 v1 稳定版，只读
- 不要 force push（约束 5）

### 用户环境

- 用户是新手，不会开 VS、不会命令行、不会配置环境
- 每次改动完编译 Release → 复制 exe + AvalonEdit.dll 到 `Serial Monitor V2/` 根目录
- Commit message 用中文，阶段性改动前后各存一个
