# Serial Monitor V2 修缮 — 开发计划

> 起点：V2 初始开发完成（92 分）。本文列出所有改进创意，按优先级分为四层。
> 每项包含：方案概要、代码量估计、涉及文件。

---

## 🔴 快速改进（Bug 修复 / 体验硬伤，优先做）

### 1. 绘图期间拖拽滑杆不卡

**根因**：OxyPlot 30Hz 刷新 + Slider.ValueChanged 每像素触发 → 同一条 UI 线程抢资源。

**方案**：检测到滑杆/摇杆被按住时，绘图降到 5Hz；松手恢复 30Hz。

```csharp
// PlotViewModel.cs 新增
public void ThrottleTo5Hz()  { _refreshInterval = 200; }  // 5Hz
public void Restore30Hz()    { _refreshInterval = 33;  }  // 30Hz
```

**代码量**：~15 行 | **难度**：小 | **文件**：`PlotViewModel.cs` + `Panels/Sliders.cs`

### 2. 发送区 HEX 非法字符实时提醒

**根因**：`HexToBytes` 静默剥离非法字符，用户发 `"AA BB GG"` 不知道 GG 被丢弃。

**方案**：发送按钮旁加隐藏 TextBlock，`ValidateHexString` 有结果时显示黄色警告。

```
"AA BB GG" → [发送]  ⚠️ 无效 HEX 字符: G
```

**代码量**：~20 行 | **难度**：极小 | **文件**：`MainWindow.xaml` + `App/MainWindow.xaml.cs`

### 3. 串口断开波形冻结保留

**根因**：关串口 → 波形清空，数据丢失。

**方案**：关串口后波形冻结，右下角叠加半透明水印"串口已断开 · 数据冻结"，重连后恢复。

**代码量**：~20 行 | **难度**：小 | **文件**：`PlotViewModel.cs` + `MainWindow.xaml`

### 4. 版本号自动注入

**方案**：csproj 加 `<Version>2.1.0</Version>`，关于页从 `Assembly.GetEntryAssembly()` 读取，不再手写。

**代码量**：~5 行 | **难度**：极小 | **文件**：`SerialMonitor.csproj` + `App/MainWindow.xaml.cs`

## 🟡 功能扩展（新增标签页 / 协议）

### 5. OLED 绘图指令

**需求**：STM32 通过串口发送绘图原语，PC 端 OLED 面板渲染。支持画点、画线、画圆、矩形、色块填充，以及小尺寸图片上传（≤64×64）。

**协议格式**：
```
[draw,point,  x,y,#RRGGBB]
[draw,line,   x1,y1,x2,y2,#RRGGBB]
[draw,rect,   x,y,w,h,#RRGGBB]
[draw,fill,   x,y,w,h,#RRGGBB]
[draw,circle, cx,cy,r,#RRGGBB]
[draw,clear]
```

**PC 端改动**：
- `MainWindow.xaml.cs`：路由加 `else if (msg.Type == "draw")`
- `Panels/Display.cs`：6 个 handler 函数，用 WPF Canvas Shape 渲染
- 协议解析器零改动（Type 字符串不校验）

**STM32 端封装**（见 `Docs/MCU/OLED绘图协议.md`）：
```c
void Draw_Line(int x1, int y1, int x2, int y2, const char *color) {
    Log("[draw,line,%d,%d,%d,%d,%s]\r\n", x1, y1, x2, y2, color);
}
// 其余 5 个同理
```

**代码量**：~150 行 | **难度**：中 | **文件**：`MainWindow.xaml.cs` + `Panels/Display.cs`

### 6. 📊 数据仪表盘

**需求**：新标签页，把四个面板的关键数据聚合在四象限 Grid 中，适合投屏/截图汇报。

```
┌──────────────┬──────────────┐
│ 最新曲线值    │ 滑杆状态条    │
│ P=0.5        │ kp ████░ 80% │
│ I=0.3        │ ki ██░░░░ 40% │
│ D=0.1        │ kd █████░ 95% │
├──────────────┼──────────────┤
│ 按键状态      │ 摇杆位置      │
│ btn1 ● ON    │ J1  X:+0.5   │
│ btn2 ○ OFF   │     Y:-0.3   │
└──────────────┴──────────────┘
```

**特点**：纯读取现有 ViewModel 数据，零协议改动，零 STM32 端代码。

**代码量**：~200 行 | **难度**：中 | **文件**：`MainWindow.xaml`（新增标签页）+ 新文件 `Panels/Dashboard.cs`

### 7. 🔬 协议调试器

**需求**：侧面板新增页，实时显示最近 20 条解析后的协议消息。DataGrid 自动滚动，类型用颜色标记。

```
时间        类型      参数
14:23:05.1  plot      P=0.500
14:23:05.1  plot      I=0.300
14:23:05.1  slider    kp=17
```

**代码量**：~100 行 | **难度**：小 | **文件**：`MainWindow.xaml` + `MainViewModel.cs`

## 🟢 打磨优化（UI 精致化 + 架构改进）

### 8. i18n 预埋——逻辑值与显示文字分家

**需求**：为未来中英双语做准备。当前致命问题：逻辑值（颜色名/风格名/发送模式）用中文存在 prefs.json，翻译会导致老用户数据丢失。

**方案**：
```
颜色:   "红色"→"red"   "绿色"→"green"   "蓝色"→"blue" ...
风格:   "手柄风"→"gamepad"   "极简风"→"minimal"   "经典风"→"classic"
发送:   "文本"→"text"   "HEX"→"hex"   "数据包"→"packet"
```

prefs.json 加载时做老中文→新 key 映射，保存时只存 key。用户零感知。

**代码量**：~80 行 | **难度**：中 | **文件**：`PreferenceService.cs` + `KeyViewModel.cs` + `KeyPanelViewModel.cs` + `SliderPanelViewModel.cs`

### 9. Segmented Control 标签

**需求**：替代侧面板 RadioButton 为 macOS 风格分段按钮——圆角胶囊 + 滑动选中指示器。

**代码量**：~100 行 | **难度**：中 | **文件**：`MainWindow.xaml`

### 10. 毛玻璃侧面板

**需求**：Win10+ `AcrylicBrush`，侧面板半透明模糊背景。

**代码量**：~20 行 | **难度**：小 | **文件**：`MainWindow.xaml`

### 11. 窗口圆角

**需求**：`WindowChrome` + `CornerRadius="8"`，现代 Windows 11 风格。

**代码量**：~10 行 | **难度**：极小 | **文件**：`MainWindow.xaml`

### 12. 面板路由从 MainWindow 抽出

**需求**：`OnPlotMessage`/`HandleKeyMessage`/`HandleSliderMessage` 等移入对应 ViewModel，MainWindow 减 400+ 行。

**代码量**：~300 行 | **难度**：中 | **文件**：`App/MainWindow.xaml.cs` + 各 ViewModel

### 13. 主题切换绑定化

**需求**：`RefreshKeysUI()` 全量重建 → `INotifyPropertyChanged` + 绑定。消除切主题闪烁。

**代码量**：~400 行 | **难度**：大 | **文件**：`Panels/Keys.cs` + `Panels/Sliders.cs` + `Panels/Joystick.cs` + `Panels/Display.cs`
