# 中英双语切换 — 设计文档

> **状态**：方案设计，待实施  
> **依赖**：v2.6 主题迁移（SetResourceReference + RegisterThemePanel + CreatePopup）  
> **设计原则**：一处注册，自动跟 —— 和 v2.6 四轮结构改进同一原则

---

## 1. 目标

- 运行时点按钮切换中/英文，和切主题亮/暗同体验——原地变化，不重建 UI，不闪烁
- 加新面板时**零额外负担**——写 UI 的中文自动就是资源 key
- 漏加英文映射不崩——缺英文处保留中文，用户一目了然

---

## 2. 核心机制

### 2.1 和主题颜色同一套 WPF 资源系统

Theme.cs 切主题做的事：

```
Resources["CardBgBrush"] = new SolidColorBrush(newColor);
  → WPF 检测资源字典值变化
    → 自动推给所有 SetResourceReference(ForegroundProperty, "CardBgBrush") 的控件
      → 控件颜色原地变化，不重建
```

双语做一模一样的事，只是值从 `SolidColorBrush` 换成 `string`：

```
Resources["清空全部"] = "Clear All";
  → WPF 检测资源字典值变化
    → 自动推给所有 SetResourceReference(ContentProperty, "清空全部") 的控件
      → 控件文字原地变化，不重建
```

`SetResourceReference` 不挑类型——WPF 属性引擎对 `Brush` 还是 `string` 是同一套传播逻辑。

### 2.2 中文即 key

UI 控件的中文直接用作资源 key。不需要另一套 key 映射表。

```
写 UI：    btn.SetResourceReference(ContentProperty, "清空全部")
切中文：   Resources["清空全部"] = "清空全部"    （恒等映射）
切英文：   Resources["清空全部"] = "Clear All"   （查英文映射表）
```

**好处**：写 UI 的人不需要去另一个文件注册 key，中文写在哪里 key 就在哪里。

## 3. API 设计

### 3.1 `LocText` 扩展方法（C# 侧）

替代所有 `Content = "中文"` / `Text = "中文"` / `Header = "中文"`：

```csharp
// Views/Locale.cs

public static class LocaleExtensions
{
    // 常用控件 — 类型安全重载
    public static void LocText(this Button btn, string zh)
        => btn.SetResourceReference(Button.ContentProperty, zh);

    public static void LocText(this TextBlock tb, string zh)
        => tb.SetResourceReference(TextBlock.TextProperty, zh);

    public static void LocText(this MenuItem mi, string zh)
        => mi.SetResourceReference(HeaderedItemsControl.HeaderProperty, zh);

    public static void LocText(this Label lb, string zh)
        => lb.SetResourceReference(Label.ContentProperty, zh);

    // 通用回退 — 任何 FrameworkElement 任意属性
    public static void LocText(this FrameworkElement fe, DependencyProperty prop, string zh)
        => fe.SetResourceReference(prop, zh);
}
```

**使用**：

```csharp
// 之前
var btn = new Button { Content = "清空全部" };
tb.Text = "确认删除 {0}？";

// 之后
var btn = new Button();   btn.LocText("清空全部");
tb.LocText("确认删除 {0}？");
```

**设计意图**：`LocText` 是唯一入口。不用它就没文字显示——和 `CreatePopup` 是唯一入口同一思路。

### 3.2 XAML 侧

继续用现有的 `{DynamicResource}`，不建新的 MarkupExtension：

```xml
<!-- 之前 -->
<Button Content="打开串口" />

<!-- 之后 -->
<Button Content="{DynamicResource 打开串口}" />
```

`DynamicResource` 和 `SetResourceReference` 底层同一套资源引用——WPF 统一推送。

### 3.3 英文映射表

一个文件，一个字典：

```csharp
// Core/LocaleData.cs

public static class LocaleData
{
    /// <summary>英文映射：中文 key → English value</summary>
    internal static readonly Dictionary<string, string> EnMap = new()
    {
        // ── 顶栏 ──
        ["打开串口"] = "Open",
        ["关闭串口"] = "Close",

        // ── 接收区 ──
        ["接收区"] = "Receive",

        // ── 通用 ──
        ["确定"] = "OK",
        ["取消"] = "Cancel",
        ["完成"] = "Done",
        ["编辑"] = "Edit",
        ["删除"] = "Delete",
        ["保存"] = "Save",
        ["返回"] = "Back",

        // ── 格式化字符串 ──
        ["确认删除 {0}？"] = "Delete {0}?",
        ["已复制: {0}"] = "Copied: {0}",
    };
}
```

**漏加英文映射的后果**：该 key 不在 EnMap 里 → 不替换 → 保留中文显示。**不崩，用户能看到并补上。**

## 4. 语言切换

```csharp
// Views/Locale.cs

public static class Locale
{
    public static string Current { get; private set; } = "zh";

    public static void SwitchTo(string lang)
    {
        Current = lang;

        if (lang == "zh")
        {
            // 所有 key 设回中文（key 本身就是中文）
            foreach (var key in LocaleData.EnMap.Keys)
                Application.Current.Resources[key] = key;
        }
        else
        {
            foreach (var (zh, en) in LocaleData.EnMap)
                Application.Current.Resources[zh] = en;
        }

        // 切完语言 → 触发条件/格式化字符串刷新
        Theme.ReapplyThemePanels();
    }
}
```

**启动初始化**：`App.xaml.cs` 或 `MainWindow` 构造函数末尾调 `Locale.SwitchTo("zh")`，把中文 key 全部注入 `Application.Current.Resources`——不注入的话 `DynamicResource` 拿不到初始值（不崩但文字不显示）。

## 5. 格式化字符串处理

带参数的字符串（"确认删除 {0}？"、"共 {0} 条"）不能直接 SetResourceReference 到显示属性——因为参数是运行时填的。

**策略**：

| 场景 | 处理方式 |
|------|---------|
| 瞬态显示（Toast、确认文字） | `string.Format(template, args)` 即时拼，不绑资源 |
| 持久显示（状态栏、标签页标题） | 少见，真需要时在 `ReapplyThemePanels` 回调里重新 Format |

切开语言时正在显示的格式化字符串极少（Toast 几秒消失），做不到完美实时跟但实际影响面极小。

## 6. 加新面板流程（含双语）

和 v2.6 已有的 8 步流程并排：

```
1.  XAML 画主 UI ─── {DynamicResource}，含文字
2.  C# 建控件 ─── LocText("中文") 替代 Content="中文"
3.  条件色 ─── RegisterThemePanel("Name", callback)
4.  Icon ─── RegisterTab(...) 一行
5.  弹窗 ─── CreatePopup + AttachPopupToggle
6.  透明按钮 ─── ApplyIconButtonTemplate
7.  复制 ─── CopyWithFeedback
8.  协议 ─── RouteMessage switch 加 case
9.  英文映射 ─── 在 LocaleData.EnMap 末尾加条目
```

**第 9 步漏了** → 该处保留中文 → 用户提 → 补一条映射。不影响功能。

## 7. 切换入口

### 7.1 位置

- **顶栏右上**：双语切换 | 主题切换 | 帮助（点击跳转 HTML）| 版本号
- **设置页侧边栏**：同功能，第二入口

两个按钮调同一个 `Locale.SwitchTo(lang)`。

### 7.2 按钮设计

固定文字 `中/EN`，切换时不改内容，改**字号和字重**。

```
中文界面： 中（大·加粗） / EN（小·常规）
英文界面： 中（小·常规） / EN（大·加粗）
```

XAML 结构：

```xml
<Button x:Name="btnLangToggleTop" Height="32"
        Style="{DynamicResource SecondaryButtonStyle}" MinWidth="0"
        Background="{DynamicResource StatusBarBgBrush}"
        BorderBrush="{DynamicResource InputBorderBrush}"
        Padding="8,0" Margin="0,0,12,0" Cursor="Hand"
        Click="BtnLangToggleTop_Click" ToolTip="切换中/English">
    <StackPanel Orientation="Horizontal">
        <TextBlock x:Name="tbLangZh" Text="中" VerticalAlignment="Center"
                   FontWeight="Bold" FontSize="14" />
        <TextBlock x:Name="tbLangSlash" Text="/" VerticalAlignment="Center"
                   FontSize="12" Foreground="{DynamicResource TextMutedBrush}" Margin="2,0" />
        <TextBlock x:Name="tbLangEn" Text="EN" VerticalAlignment="Center"
                   FontWeight="Regular" FontSize="11" />
    </StackPanel>
</Button>
```

切语言时更新按钮（在 `Locale.SwitchTo` 内调，或事件直接设）：

```csharp
void UpdateLangButton(bool isZh)
{
    if (isZh)
    {
        tbLangZh.FontWeight = FontWeights.Bold;   tbLangZh.FontSize = 14;
        tbLangEn.FontWeight = FontWeights.Regular; tbLangEn.FontSize = 11;
    }
    else
    {
        tbLangZh.FontWeight = FontWeights.Regular; tbLangZh.FontSize = 11;
        tbLangEn.FontWeight = FontWeights.Bold;    tbLangEn.FontSize = 14;
    }
}
```

### 7.3 风格

按钮背景/边框跟顶栏主题按钮一致：`StatusBarBgBrush` + `InputBorderBrush` + `SecondaryButtonStyle`。切主题时颜色自动跟——3 个 TextBlock 的 Foreground 不硬编码，靠父级 `SecondaryButtonStyle` 继承。

## 8. 技术边界（不做的事）

| 不做 | 原因 | 后果 |
|------|------|------|
| AvalonEdit SearchPanel 翻译 | 库内部 UI，不能绑 WPF 资源 | 搜索框永远英文 |
| MessageBox 按钮翻译 | Win32 原生对话框，跟 Windows 系统语言 | "确定/取消" 跟系统不跟应用 |
| 重排布局 | 中文→英文文字长度变化，极少数按钮可能变窄/溢出 | 给关键按钮设 MinWidth，或不做——英文通常比中文短 |
| 自动化测试 | WPF UI 字符串没有自动化测试框架 | 切一次语言 + 7 面板逐个过 = 5 分钟肉眼验证 |

## 9. 必避坑：旧坑在双语场景的对应形态

> 来源：`../Serial V2 Initial/当前状态.md`（Phase 1-6 27 条约束 + BAML 缓存 + icon.ico 7h）  
> 来源：`当前状态2.md`（v2.6 主题迁移 V1-V12）

### 9.1 颜色回调踩文字（V6/V7 对应）

**原坑**：主题迁移时 Keys 回调 `foreach (tb in titleBar.Children)` 无差别覆盖所有 TextBlock → modeTag.Foreground 被误刷为 TextMutedBrush。

**双语对应**：切语言后 `ReapplyThemePanels` 跑条件色回调。回调里如果遍历 `TextBlock.Text` 并赋值，会直接摧毁 `LocText` 绑的 `SetResourceReference`——文字被硬编码覆盖，下次切语言不跟。

**防法**：条件色回调只碰颜色属性（Foreground / Background / BorderBrush）。文字属性只走 `LocText` / `DynamicResource`。互不踩脚。

### 9.2 硬编码不跟主题（V8-V10 对应）

**原坑**：Sensor 组背景色 `#1C1C1E` / 进度条轨道色硬编码 hex → 切主题不动。

**双语对应**：字符串硬编码 `Content = "清空全部"` → 切语言不动。和主题迁移时硬编码 hex 同一模式——不走入口就不跟。

**防法**：`LocText` 是唯一入口。不走它就没文字。遗漏暴露为"切语言后该处仍显示中文"——不崩但一眼可见。

### 9.3 BAML 缓存掩盖 + 重编崩溃（icon.ico 7h / Phase 6 B1 对应）

**原坑**：移动 `icon.ico` 后 XAML 路径没改 → 旧 BAML 缓存掩盖 → 下次重编崩溃。新增 `.xaml` 资源文件同风险。

**双语对应**：方案 B 不建 `.xaml` / `.resx` 文件——语言数据纯 C# 字典。**不触发这个风险。**

### 9.4 Popup 独立视觉树（约束 #24/#25）

**原坑**：代码创建的 ContextMenu/Popup 不加 `PlacementTarget` → 找不到主窗口的 `DynamicResource` → 暗色白底白字。

**双语对应**：字符串存在 `Application.Current.Resources` 级别——Popup 查资源链能查到。**和颜色不同，不要求 PlacementTarget。** 但如果未来有人在 Popup 内部新建了独立 `ResourceDictionary`，就会切断查找链。

**防法**：字符串资源不放 Popup 内部。始终走 Application 级别。

### 9.5 第三方库 UI 不可控（AvalonEdit / MessageBox）

**原坑**：AvalonEdit SearchPanel 是库内部 UI，`ContextMenuMenuItemStyle` 管不到。MessageBox 是 Win32，不走 WPF 视觉树。

**双语对应**：同上——这两个地方永远不跟语言切换。已在 §8 标注为不做。

## 11. 和 v2.6 体系的对应关系

| v2.6 基础设施 | 双语对应 |
|---------------|---------|
| `SetResourceReference(Property, "CardBgBrush")` | `LocText("中文")` — 同一个 API |
| `{DynamicResource CardBgBrush}` in XAML | `{DynamicResource 中文key}` — 同一个标记 |
| Theme.cs `Resources[key] = newValue` foreach 21 色 | Locale.SwitchTo `Resources[key] = value` foreach ~100 字符串 |
| `RegisterThemePanel` 条件色回调 | `Theme.ReapplyThemePanels()` 切语言后刷新格式化字符串 |
| `CreatePopup` 唯一弹窗入口 | `LocText` 唯一文字入口 |
| 93 处 FindResource → SetResourceReference | ~100 处内联字符串 → LocText/DynamicResource |
| 净减 ~200 行样板 | 净增英文映射 ~100 条（无法避免的翻译数据） |

## 12. 评估

### 优势

- **零新概念**：和切主题颜色完全同一个机制，项目里已验证
- **不闪不重建**：WPF 资源属性引擎自动推，控件不动
- **漏英文不崩**：缺英文保留中文，用户可见可补
- **加面板无负担**：写 UI 时中文直接就是 key
- **和 v2.6 基础设施无缝衔接**：`LocText` 走 `SetResourceReference`，和颜色同一套

### 风险

- 格式化字符串切语言时做不到完美实时跟（影响面极小——瞬态文本）
- MessageBox/AvalonEdit 不跟（第三方/系统限制）
- 100 处替换只能肉眼验证

### 遗留尾巴

- 以后可以替换 MessageBox 为 WPF 自定义 Dialog → 文字可控
- 以后可以给格式化字符串设计专门的 `LocFormatText` 重载 → 切语言自动重算
