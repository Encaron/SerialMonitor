## 前情提要

项目始于江科大 WinForms 串口助手——用户改了三个默认值（115200 / 文本 / UTF-8），从此开启了"工具应该记住我怎么用"的基因。

**v1.0.0**（WPF 重写 UI，30+ 功能）→ **V2**（`[type,arg,...]` 协议系统 + OxyPlot 波形 + 4 控制面板 + .NET 8）。Phase 1-6 共 ~100 commits，踩坑无数（OxyPlot 不能写 XAML、ContextMenu 暗色失效、icon.ico 移动引发 7h 崩溃……），全部记录在 `../Serial V2 Initial/当前状态.md` 和 memory 中。

用户是嵌入式开发者（STM32/ESP32），不会 C#/WPF，只双击 exe。技术栈：WPF .NET 8 + OxyPlot 2.2 + AvalonEdit + 21 色双主题。编译 `dotnet publish -c Release`，产出 1.95MB 单文件 exe。已发布 GitHub v2.1.0。

## #8 调参工作台 — 踩坑记录

> 5 个文件，+323 / -20 行，~4 小时（含排查）

| # | 现象 | 根因 | 修法 |
|:--:|------|------|------|
| B1 | 打开抽屉瞬间 OxyPlot `NullReferenceException` 崩溃 | `InvalidatePlot(true)` 走 Background dispatch，与抽屉展开时 plotArea resize 触发的 OxyPlot 内部渲染同时跑，`GetClippingRect` 拿到空引用 | 三管齐下：① `InvalidatePlot(false)` 不强制同步渲染 ② `_backgroundRenderScheduled` 防重入锁 ③ `panelPlot.UpdateLayout()` 先完成布局再改 Visibility |
| B2 | Sliders 页滑杆拖不动 | `HandleSliderMessage` 每条 `[slider,...]` 调 `InitSliderPanel()` → 现在每次都跑 `RefreshSlidersUI()` 销毁重建，拖拽中控件被回收 | `InitSliderPanel(bool refreshUI=true)`，协议路径传 `false`，只确保 VM 存在不重建 UI |
| B3 | Sliders 页进去看不到滑杆 | `EnsureSliderVM()` 提前创建了 `_sliderVM` → `InitSliderPanel()` 的 `if (_sliderVM != null) return;` 把 UI 创建也跳过了 | 拆开 VM 创建和 UI 刷新：VM 只建一次，UI 每次 `refreshUI=true` 都跑 |
| B4 | 在 Plot 页开串口，调参按钮不出现 | `RefreshTuningDrawer()` 只在切标签页时调用，已在 Plot 页时没有触发点 | `[plot,...]` 和 `[slider,...]` 到达时，如果按钮还没显示且当前在 Plot 页，调一次 `RefreshTuningDrawer()` |
| B5 | 切出 Plot 页再切回，波形冻结不更新 | 非 Plot 期间数据照存但跳渲染（`IsActive=false`），切回来时 `_dirty=true` 没 flush，等下一次数据才渲 | 检测 `IsActive` false→true 跳变时调 `Flush()` |
| B6 | 调参按钮遮挡 X 轴时间标签 | 按钮在 `plotArea` 内部 bottom-center 浮层，和 OxyPlot 横轴标签重叠 | 移出 plotArea，单独一行（波形图和 GridSplitter 之间），140×24 PrimaryButtonStyle |

### 架构备忘

- **VM 共享**：调参抽屉的滑杆和 Sliders 页共用 `_sliderVM.Sliders[n]`，不是拷贝。步长/颜色/范围双向即时生效。
- **名字交集**：`_plotVM.GetChannelNames() ∩ _sliderVM.Sliders` → 只显示有波形曲线的滑杆
- **调度法**：`UseBackgroundRender=true` 时 `InvalidatePlot(false)` 走 `DispatcherPriority.Background`，让路给 MouseMove。拖拽流畅，波形帧率自然降到 15-25Hz，松手恢复满 30Hz
- **波形压扁不是 Bug**：抽屉弹出后绘图区物理高度减小，曲线变扁但数据不变。Y 轴 Zoom、CSV 导出、HUD 数值全部不受影响
- **`EnsureSliderVM()`**：轻量 VM 初始化（不建 UI），供调参抽屉在 Sliders 页未访问时使用。从 prefs.json 恢复已保存的滑杆配置

## #10 FFT 频谱 — 踩坑记录

> ~12 小时（含 3 步迭代 + UI 打磨），6 个文件，+550 / -50 行

| # | 现象 | 根因 | 修法 |
|:--:|------|------|------|
| B1 | 启动即崩溃，`TargetInvocationException` → `NullReferenceException at chkFreqLines_Changed` | `IsChecked="True"` 写在 XAML 里，BAML 加载触发 Checked 事件，`_plotVM` 尚为 null | XAML 去掉 `IsChecked`，构造函数里 `chkFreqLines.IsChecked = true`（约束 19） |
| B2 | 数据源下拉菜单闪烁 | `[fft,...]` 每帧调 `RefreshFreqSourceList()` → `Items.Clear()` 重建 | 改为 `DropDownOpened` 刷新 + 新项直加 `Items.Add()` |
| B3 | 切回 plot 源后频谱一条直线 | `SetFftSource("plot:Slider1")` 里 `_fftBuffer = new double[128]` 全零 → FFT(零)=零 | 只在大小时变时重建 buffer，不丢旧数据 |
| B4 | 频域下 HUD 仍在更新 | `OnPlotMessage` 无条件调 `UpdatePlotHud()` | 加 `if (!_isFreqDomain)` 守卫 |
| B5 | 窗函数下拉两个"汉宁" | 两次初始化代码重复（#10 FFT 窗函数下拉框 + #10 FFT: 窗函数） | 合并去重 |
| B6 | 11 个僵尸进程锁住 exe | IsChecked BAML 崩溃后窗口未创建但进程存活，用户双击 11 次 → 11 个死进程 | 任务管理器杀进程；根因已修 B1 |

### 架构备忘 + 创意决策

- **双 Model 单视图**：`TimeModel` + `FreqModel`，`plotView.Model` 换引用。切换后各自保留缩放/滚动状态
- **PC 端自动 FFT**：`[plot,...]` 数据滑窗 buffer（128 点默认）→ 自研 radix-2 FFT（纯 C# ~50 行，零依赖）→ 实时频谱。用户 STM32 零改动就能看频域
- **`[fft,name,点数,bin0,...]` 命名协议**：给 FFT 加名字字段，兼容旧格式。数据源下拉框分 `📈 plot` 和 `📶 fft` 两类，选中命名 FFT 后不被自动 FFT 覆盖
- **数据源蓝色边框卡片**：`PrimaryBrush` 描边 + 15px 标签 + ↻ 刷新按钮（22×22），上下层布局防拥挤
- **采样率 → Hz**：侧面板加采样率输入框 → X 轴从 Bin 索引变 Hz，频率范围/分辨率/基频全部 Hz 显示
- **窗函数生效**：汉宁/矩形/汉明/布莱克曼四选一，`RecomputeFft` 内乘入 buffer。CMSIS-DSP 同名对齐
- **频域 HUD**：右上角 `PrimaryBrush` 边框标签，显示当前数据源名（`📈 Slider1` 或 `📶 ch1`），时域则走原有数值 HUD
- **PerspectiveToggleStyle**：Ghost 描边（`PrimaryBrush` 边框/文字 + 透明底），介于 tab 和按钮之间。按钮文案 `⏱ 时域 → 频域 📶` / `📶 频域 → 时域 ⏱`，箭头指向目标
- **暂停冻结两路**：`IsPaused` 同时冻结 `[plot,...]` 和 `[fft,...]`，切换视角只看不倒数据

## v2.6.0 — 踩坑记录

> 2026-06-21，三项内部优化：i18n 预埋 + 面板路由抽出 + 主题切换优化。3 个文件，~300 行改动。

| # | 现象 | 根因 | 修法 |
|:--:|------|------|------|
| **V1** | #11 ComboBox 存匿名对象 → 读回 `ToString()` 拿到类名而非显示文字 | `cb.SelectedItem = new { Label, Value }`，WPF 用反射读 `DisplayMemberPath`，匿名对象 internal 属性反射不可靠 | 回退方案：ComboBox 存中文字符串，读写时走 `MigrateSendMode`/`DisplaySendMode` 映射。`cb.SelectedItem = LogicValueMaps.DisplaySendMode(key)` |

| **V5** | #16 拖拽中切主题，滑杆自定义色滑块进度条差一个色值 | 拖拽期间颜色更新走自身迷你逻辑，切主题只刷默认色不刷自定义色的拖拽态 | 已知残余 Bug，不严重——松手恢复。改需拖拽态也响应主题切换（低优先级） |

### 架构备忘

- **i18n 映射集中管理**：`LogicValueMaps.cs` 3 组双向 `Dictionary<string,string>` + `MigrateXxx` + `DisplayXxx`。新增逻辑值只加字典不改代码。
- **RouteMessage 活接头**：switch 内部是 10 个 `case` → 直调已有 handler。未来想升级为 `Dictionary<string, Action>` 注册表，只改 switch 内部一行不动 case。
- **UpdateXxxTheme 遍历换色**：不 `Clear` + 重建，改 Brush 引用。WPF Freezable 对象共享，改引用不创建新实例，开销极低。
- **6 个已有 handler 不动**：key/slider/joystick 在各自 partial class（Keys.cs/Sliders.cs/Joystick.cs），fft 在 MainWindow，display/draw 在 Display.cs。搬进 VM 要传 5-8 个回调参数，比原地提取更复杂。

## v2.6 后半段：从补漏到改结构（2026-06-21）

> 基线 `b5770a2`。v2.6.0 三任务完成后，用户追问"为什么同一个 bug 反复出现"，
> 引出一轮架构反思：不是人粗心，是代码结构逼人粗心。

### 核心发现

所有高频遗漏都是同一个根：**一条规则散在 N 处手写，忘一处就漏。**

| 遗漏现象 | 散在几处 | 机制 |
|------|:--:|------|
| 切主题颜色不跟 | 267 处 FindResource + 4 个 UpdateXxxTheme | 人记 |
| 复制忘 Q 弹动画 | 14 对 SafeSetClipboard + ShowCopyToastAndShake | 人拼 |
| 下拉菜单白底白字 | 7 处 new ContextMenu | 人记 |
| Popup 暗色炸 | 5 处 new Popup | 人记 |
| 新建图标不会抖/不切面板 | 每图标 5 处 (XAML/Checked/Visible/Animation/Handler) | 人记 |

### 解法：一处注册，自动跟

加新功能只在一处声明"这是什么"，框架自动处理重复部分。

### 四轮改进

| 轮 | 做了什么 | 之前 | 之后 | commit |

共碰 14 个文件，净减 ~10 行。0 错误 0 警告，72 条测试全绿。

## v2.6 后半段：主题迁移（B+C）✅ 全部完成（2026-06-21）

### 背景

#16 的 4 个 `UpdateXxxTheme` 兜底 95%。但加新面板仍需人去 `UpdateXxxTheme` 加遍历——和四轮结构改进同病："散在 N 处手写，忘一处就漏"。

### 方案

- **B（SetResourceReference）**：纯色属性 → `SetResourceReference(Property, "ResourceKey")`，WPF 内置自动跟，零回调代码
- **C（RegisterThemePanel）**：条件色（编辑态蓝条/拖拽差色/硬编码 hex）→ 面板级回调，切主题时统一执行

### 成果

| 维度 | 之前 | 之后 |
|------|------|------|
| 面板主题回调 | 5 硬编码 `if (_xxxVM) UpdateXxxTheme()` 在 Theme.cs | 0——`foreach (var cb in _themePanelCallbacks.Values) cb()` |
| UpdateXxxTheme 方法 | 4 个（Keys/Sliders/Joystick/Display） | 0——全部删除 |
| SetResourceReference | 0 处 | **93 处** 跨 6 文件（Keys/Sliders/Joystick/Display/Sensors/QuickSend） |
| Sensors 主题更新 | `RefreshAllRows()` 全量重建 | RegisterThemePanel 回调（纯色 SetResourceReference + 条件色遍历） |
| 弹窗 | 3 处 `staysOpen: true` + 各自补丁 | `CreatePopup` 统一内置 StaysOpen=false + LocationChanged |
| 弹窗按钮 toggle | 手动 PreviewMouseLeftButtonDown 补丁 | `AttachPopupToggle` 统一 |
| 透明按钮模板 | 8 处漏加 → 暗色浅蓝底框 | `ApplyIconButtonTemplate` 全加 + 提至 Theme.cs |
| 切主题体感 | Sensors 全量重建闪烁 | 6 面板全部原地变色，不闪不跳不重建 |

### 踩坑

| # | 现象 | 根因 | 修法 |
|:--:|------|------|------|
| V6 | modeTag.Foreground 切主题变 TextMutedBrush | Keys 回调 `foreach (tb in titleBar.Children)` 不区分 nameLabel/modeTag | `Children[0]` 精确索引，modeTag 走 SetResourceReference |
| V7 | valTb（滑杆值显示）颜色不跟 | 回调 `UpdateSlidersGridTheme` 遍历所有 TextBlock 覆盖 | 只更新 Row 0 nameLabel，valTb 走 SetResourceReference |
| V8 | Sensor 组背景色不跟 | `#1C1C1E/#E6E6EB` 硬编码 hex 不在 ThemeMap | 进 RegisterThemePanel 回调遍历更新 |
| V9 | 进度条轨道/开关 off 态不跟 | 同类硬编码 hex | 进回调 + 加 `_cardProgressTrackMap` 引用 |

| V12 | 弹窗再点按钮闪一下重开 | `StaysOpen=false` 在 Click 前关 Popup | `AttachPopupToggle` 用 PreviewMouseLeftButtonDown 拦截 |

已加入 `memory/hard-constraints.md`：
- #34 弹窗必须走 CreatePopup

### 加新面板/新 UI 的流程

```
1. XAML 画主 UI → {DynamicResource} 天然安全
2. C# 建控件 → SetResourceReference（不写 FindResource）
3. 条件色 → RegisterThemePanel("Name", callback) 一行
4. Icon → RegisterTab 一行
5. 弹窗 → CreatePopup + AttachPopupToggle 各一行
6. 透明按钮 → ApplyIconButtonTemplate 一行
7. 复制 → CopyWithFeedback 一行
8. 协议 → RouteMessage switch 加 case
```
