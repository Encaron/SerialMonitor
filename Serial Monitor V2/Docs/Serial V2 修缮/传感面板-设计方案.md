<a id="一协议格式已定"></a>
## 一、协议格式（已定）

### MCU → PC：传感数据上报

```
[sensor,子类型,卡片名,值,辅助参数]
```

| 子类型 | 示例 | 值 | 辅助参数 |
|--------|------|----|----------|
| `temp` | `[sensor,temp,芯片温度,42.5,45.0]` | 温度值 | 可选（PWM 占空比等） |
| `humidity` | `[sensor,humidity,ambient,68.2]` | 湿度 % | 无 |
| `pressure` | `[sensor,pressure,大气压,1013.2]` | 气压值 hPa | 无需——PC 端自算百分比 |
| `status` | `[sensor,status,主板,online]` | 状态字 | 可选描述 |
| `status` | `[sensor,status,主板,alarm,overheat]` | 告警 | 告警原因 |
| `motor` | `[sensor,motor,电机1,2450,85]` | 转速 RPM | PWM 占空比%（可选） |
| `slider` | `[sensor,slider,电机1目标,2450]` | 当前值 | PC 端发 `[ctrl,slider,...]`，MCU 回此确认 |
| `generic` | `[sensor,generic,电池电压,3.72,85]` | 任意值 | 辅助参数可选。前两字段固定 `sensor,generic` |

### PC → MCU：开关控制

```
[ctrl,子类型,卡片名,动作]
```

| 子类型 | 示例 | 动作 |
|--------|------|------|
| `led` | `[ctrl,led,蓝色LED,on]` | on / off |
| `relay` | `[ctrl,relay,电源,off]` | on / off |
| `slider` | `[ctrl,slider,电机1目标,2450]` | 滑块当前值 |

### 名字即显示名

`[sensor,temp,最近温度,42.0]` → 卡片标题显示"最近温度"。固件端写什么 PC 端就显示什么，中英随你。

### 解析规则

**逗号是唯一分隔符。** `ProtocolParser` 不推断字段语义，只数逗号、定位置。

| 规则 | 说明 |
|------|------|
| 字段顺序固定 | `[type, 子类型, 名字, 值, 辅助]`，字段不可换位 |
| 末尾字段可省略 | `[sensor,humidity,ambient,68.2]` → Args 共 4 个，辅助 = null |
| 中段留空用双逗号 | `[sensor,temp,温度,,45.0]` → 值 = ""，辅助 = "45.0"。`` 被正确保留为空字符串 |
| 名字不可含逗号 | `[sensor,temp,芯,片,42.5]` → 被切为 6 个字段，全串位 |
| 数值中含小数点不受影响 | `42.5` 是一个字段，`.` 不是分隔符 |

**中段留空示例**：传感器读回异常时固件可以只发辅助参数：

```c
// 正常：两个值都有
Serial_Printf("[sensor,temp,温度,%.1f,%.0f]\r\n", temp_value, duty_pct);
// → [sensor,temp,温度,35.0,45]

// 温度值失效，只发辅助参数（双逗号留空）
Serial_Printf("[sensor,temp,温度,,%.0f]\r\n", duty_pct);
// → [sensor,temp,温度,,45]
```

PC 端 `Update()` 中：
- `value == ""` → 卡片主值显示 `--`，不进入 History（不画波形）
- `aux != null` → 卡片辅助行照常显示

### SensorGroup（组）

```csharp
/// <summary>
/// 一个组 = 一个灰色圆角框。组内卡片按 `"---"` 换行标记拆分为多个排（WrapPanel）。
/// </summary>
public class SensorGroup
{
    /// <summary>组名（用户命名）。null 或空 = 不显示标签。</summary>
    public string Name { get; set; }
    
    /// <summary>组内的卡片和换行标记。`"---"` 字符串 = 排间换行。</summary>
    public ObservableCollection<object> Items { get; set; } = new();
    
    // 便捷方法
    public IEnumerable<SensorCardViewModel> Cards
        => Items.OfType<SensorCardViewModel>();
    
    public void AddCard(SensorCardViewModel card) => Items.Add(card);
    public void AddLineBreak() => Items.Add("---");
    public void RemoveItem(object item) => Items.Remove(item);
    
    // -- 序列化 --
    public Dictionary<string, object> ToDict() => new()
    {
        ["name"] = Name ?? "",
        ["items"] = Items.Select(item =>
        {
            if (item is SensorCardViewModel card)
                return (object)card.ToDict();
            return "---";  // 换行标记
        }).ToList(),
    };
    
    public static SensorGroup FromDict(Dictionary<string, object> d)
    {
        var group = new SensorGroup 
        { 
            Name = d.GetValueOrDefault("name")?.ToString() ?? "" 
        };
        if (d.TryGetValue("items", out var itemsObj) 
            && itemsObj is List<object> itemList)
        {
            foreach (var item in itemList)
            {
                if (item is string s && s == "---")
                    group.Items.Add("---");
                else if (item is Dictionary<string, object> cd)
                    group.Items.Add(SensorCardViewModel.FromDict(cd));
            }
        }
        return row;
    }
}
```

### SensorPanelViewModel

```csharp
public class SensorPanelViewModel
{
    public string Title { get; set; } = "传感面板";
    public ObservableCollection<SensorGroup> Groups { get; } = new();
    public bool IsActive { get; set; } = true;
    public HashSet<string> DeletedNames { get; } = new();
    public bool IsEditMode { get; set; }
    
    // -- 组操作 --
    public SensorGroup AddGroup(string name = null)
    {
        var group = new SensorGroup { Name = name ?? "" };
        Rows.Add(row);
        return row;
    }
    
    public void RemoveGroup(SensorGroup group) => Rows.Remove(row);
    
    // -- 卡片操作 --
    public void AddCardToGroup(SensorGroup group, string type, string name)
    {
        group.Items.Add(new SensorCardViewModel { Type = type, Name = name });
    }
    
    public void RemoveCard(SensorCardViewModel card)
    {
        DeletedNames.Add(card.Name);
        foreach (var group in Rows)
            group.Items.Remove(card);
    }
    
    // -- 数据路由 --
    public void OnSensorMessage(string subType, string name, 
        string value, string aux)
    {
        if (DeletedNames.Contains(name)) return;
        
        var card = FindByName(name);
        if (card == null)
        {
            if (Rows.Count == 0) AddGroup();
            card = new SensorCardViewModel { Type = subType, Name = name };
            Rows[^1].Items.Add(card);
        }
        card.Update(value, aux);
    }
    
    public SensorCardViewModel FindByName(string name)
        => Rows.SelectMany(r => r.Cards).FirstOrDefault(c => c.Name == name);
    
    // -- 序列化 --
    public List<Dictionary<string, object>> Serialize()
        => Rows.Select(r => r.ToDict()).ToList();
    
    public void Deserialize(List<Dictionary<string, object>> data)
    {
        Rows.Clear();
        if (data == null) return;
        foreach (var d in data)
            Rows.Add(SensorGroup.FromDict(d));
    }
}
```

### SensorCardViewModel

```csharp
public class SensorCardViewModel
{
    public string Type { get; set; }          // 英文 key: "temp"/"humidity"/"pressure"/"status"/"control"
    public string Name { get; set; }          // 显示名: "芯片温度"
    public string Value { get; set; } = "--"; // 当前值
    public string AuxText { get; set; }       // 辅助参数（无则空）
    public string Status { get; set; }        // 仅 status 类: "online"/"alarm"/"offline"
    public string AlarmReason { get; set; }   // 告警原因
    public FixedSizeQueue<double> History { get; } = new(30);

    /// <summary>数值单位。status/control 无单位返回 ""。</summary>
    public string GetUnit() => Type switch
    {
        "temp"     => "°C",
        "humidity" => "%",
        "pressure" => "hPa",
        "motor"    => " RPM",
        "slider"   => "",
        _          => "",
    };

    // 卡片颜色（C# 算，不走 XAML 绑定）
    public string GetAccentHex(bool isDark)
    {
        return Type switch
        {
            "temp"     => isDark ? "#FFC107" : "#F5A623",
            "humidity" => isDark ? "#42A5F5" : "#2196F3",
            "pressure" => isDark ? "#26C6DA" : "#00BCD4",
            "motor"    => isDark ? "#AB47BC" : "#9C27B0",
            "slider"   => isDark ? "#5C6BC0" : "#3F51B5",
            "generic"  => ColorHex ?? (isDark ? "#78909C" : "#607D8B"),
            "control"  => isDark ? "#FFA726" : "#FF9800",
            "status"   => Status switch
            {
                "alarm"   => isDark ? "#EF5350" : "#F44336",
                "offline" => isDark ? "#EF5350" : "#F44336",
                _         => isDark ? "#66BB6A" : "#4CAF50",
            },
            _ => isDark ? "#B0B0B0" : "#888888",
        };
    }

    public void Update(string value, string aux)
    {
        if (Type == "status")
        {
            Status = value;
            Value = string.IsNullOrEmpty(value) ? "--" : value;
            AlarmReason = aux;
        }
        else
        {
            // 空值 → 显示 "--"，不进 History
            Value = string.IsNullOrEmpty(value) ? "--" : value;
            AuxText = aux;
            if (double.TryParse(value, NumberStyles.Float, 
                CultureInfo.InvariantCulture, out double v))
                History.Enqueue(v);
        }
    }

    public Dictionary<string, object> ToDict() => new()
    {
        ["type"] = Type,
        ["name"] = Name,
    };

    public static SensorCardViewModel FromDict(Dictionary<string, object> d)
    {
        if (d == null) return null;
        return new SensorCardViewModel
        {
            Type = d.GetValueOrDefault("type")?.ToString() ?? "temp",
            Name = d.GetValueOrDefault("name")?.ToString() ?? "Untitled",
        };
    }
}
```

## 四、卡片模板 × 8 类（已定）

### 通用结构

```
┌─ Border (圆角 8px, Width=160, Height=200, 固定尺寸) ────┐
│ ┌────────────────────┬──────────────────────────────────┐ │
│ │ 左竖条 12px         │  StackPanel 垂直分层               │ │
│ │ 固定色              │  标题（卡片名）12pt                 │ │
│ │                    │  数值 20pt Bold 同竖条色 + 单位    │ │
│ │                    │  辅助文字 9pt 浅灰（无可不显）        │ │
│ │                    │  迷你波形 Canvas Polygon 面积图     │ │
│ └────────────────────┴──────────────────────────────────┘ │
└──────────────────────────────────────────────────────────┘
```

**共性**：高 200px、宽 160px、左竖条 12px 贯穿全高、圆角 8px。竖条色 = 主值色 = 波形色（三处同源 `GetAccentHex()`）。亮色白底暗色深灰底，切主题自动跟。单位由 `GetUnit()` 返回，渲染时追加到大数字后。

**波形**：面积图（Polygon），折线到基线之间填满 30% 透明度的竖条色。线条 1px 保持精细，填充面积增强视觉重量。30 点，无坐标轴/刻度/图例。

```csharp
private Border CreateSensorCard(SensorCardViewModel vm)
{
    var accentHex = vm.GetAccentHex(isDarkTheme);
    var accentBrush = ParseColor(accentHex);
    var accentFill = ParseColor(accentHex, alpha: 0.30); // 波形填充 30% 透明

    var strip = new Rectangle { Width = 12, Fill = accentBrush };

    // 标题
    var title = new TextBlock 
    { 
        Text = vm.Name,
        FontSize = 12,
        Foreground = (Brush)FindResource("TextPrimaryBrush"),
    };

    // 数值（status 显状态字，sensor 类显值+单位，control 显 ON/OFF）
    string displayValue = vm.Type switch
    {
        "status"  => vm.Status ?? "--",
        "control" => vm.Value ?? "--",
        _         => (vm.Value ?? "--") + vm.GetUnit(),  // "42.5" → "42.5°C"
    };

    { 
        Text = displayValue,
        FontSize = 20, FontWeight = FontWeights.Bold,

    };

    };

    // 进度条（仅 humidity / pressure）
    FrameworkElement progressBar = null;
    if (vm.Type == "humidity" || vm.Type == "pressure")
    {
        double pct = vm.Type == "pressure"
            ? (double.TryParse(vm.Value, out double hpa) ? hpa / 1013.25 * 100 : 0)
            : (double.TryParse(vm.Value, out double hum) ? hum : 0);
        progressBar = CreateProgressBar(pct, accentBrush);

    // 迷你波形（sensor 类有，status/control 无）。面积图 = Polygon
    FrameworkElement miniPlot = null;

        miniPlot = CreateMiniPlot(vm, accentBrush, accentFill);

    // 内容 StackPanel
    var stack = new StackPanel { Margin = new Thickness(8, 6, 8, 6) };
    stack.Children.Add(title);
    stack.Children.Add(valueText);
    if (progressBar != null)
    {
        stack.Children.Add(progressBar);
        // 湿度/气压不再加 auxText——进度条已包含百分比信息
    }
    else
    {
        stack.Children.Add(auxText);
    }
    if (miniPlot != null) stack.Children.Add(miniPlot);

    // control 类加 ToggleButton
    if (vm.Type == "control")

        var toggle = CreateToggleButton(vm); // ⬜ 按钮具体样式待讨论 D

        BorderThickness = new Thickness(0.5),
        CornerRadius = new CornerRadius(8),
        Child = grid,
    };
```

## 五、迷你波形（已定）

**面积图 = Polygon 填满**。折线上每个点，从底部（y=基线）往回兜一圈封闭，中间填 30% 透明度竖条色。线条 1px 保持精细轮廓，填充面积增强视觉重量——小卡片上不会显得单薄，也不会因加粗线条而损失波形细节。

Canvas 自绘，无坐标轴，无刻度，无图例。

```csharp
/// <summary>
/// 创建迷你波形 Canvas。内部含一个 Polygon（面积填充）+ 一个 Polyline（轮廓线）。
/// </summary>
private Canvas CreateMiniPlot(SensorCardViewModel vm, Brush lineBrush, Brush fillBrush)
{
    var canvas = new Canvas 
    { 
        Width = 140, Height = 40,
        Margin = new Thickness(0, 4, 0, 0),
    };

    // 填充面积（先加，在下层）
    var polygon = new Polygon
    {
        Fill = fillBrush,           // 竖条色 + 30% 透明度
        Stroke = null,              // 填充区无边框
    };

    // 轮廓线（后加，在上层）
    var polyline = new Polyline
    {
        Stroke = lineBrush,
        StrokeThickness = 1.0,
        StrokeLineJoin = PenLineJoin.Round,
    };

    canvas.Children.Add(polygon);
    canvas.Children.Add(polyline);
    _miniPlotMap[vm] = (polygon, polyline);
    return canvas;
}

// 30Hz 批量刷新
private void RefreshMiniPlots()
{
    if (!_sensorVM.IsActive) return; // 切走了不渲染

    foreach (var (vm, (polygon, polyline)) in _miniPlotMap)
    {
        var data = vm.History.GetSnapshot();
        polygon.Points.Clear();
        polyline.Points.Clear();
        if (data.Count < 2) continue;

        double canvasW = 140, canvasH = 40;
        double yMin = data.Min(), yMax = data.Max();
        double yRange = (yMax - yMin) > 0.1 ? (yMax - yMin) : 1.0;

        // 折线上点
        var linePoints = new PointCollection();
        for (int i = 0; i < data.Count; i++)
        {
            double x = canvasW * i / (data.Count - 1);
            double y = canvasH * (1.0 - (data[i] - yMin) / yRange * 0.8);
            linePoints.Add(new Point(x, y));
        }
        polyline.Points = linePoints;

        // 面积图：折线点 + 左下角 + 右下角 → 封闭多边形
        var areaPoints = new PointCollection(linePoints);
        areaPoints.Add(new Point(canvasW, canvasH));  // 右下角
        areaPoints.Add(new Point(0, canvasH));         // 左下角
        polygon.Points = areaPoints;
    }
}
```

**为何面积图不糊**：1px 线条保留峰谷细节，填充面积提供视觉重量。二者互补——线条让你看清波形走向，填充让你一眼看到卡片群中有波形在动。加粗线条 2px+ 会在 140px 宽画布上抹平相邻点差异，面积图方案不碰线宽。

**防卡链**：`IsActive = false`（切到其他标签页）→ `RefreshMiniPlots` 直接 return → 零渲染开销。数据路径不断——`OnSensorMessage` 照跑、History 照存。切回来下一帧自然刷新。

```csharp
private StackPanel _rowsStackPanel; // 所有行的容器

private void RefreshAllRows()
{
    _rowsStackPanel.Children.Clear();
    _miniPlotMap.Clear();
    
    foreach (var group in _sensorVM.Groups)
    {
        // 组——Border 包裹，统一背景色，行间间距 12px
        var groupBorder = new Border
        {
            Background = (Brush)FindResource("SecondaryHoverBgBrush"),
            CornerRadius = new CornerRadius(6),
            Padding = new Thickness(8, 6, 8, 8),
            Margin = new Thickness(0, 0, 0, 12),
        };
        
        var groupPanel = new StackPanel();
        
        // 组名标签（有则显）
        if (!string.IsNullOrEmpty(group.Name))
        {
            groupPanel.Children.Add(new TextBlock
            {
                Text = group.Name,
                FontSize = 11,
                Foreground = (Brush)FindResource("TextMutedBrush"),
                Margin = new Thickness(4, 0, 0, 4),
            });
        }
        
        // 渲染该行——按 "---" 换行标记拆成多个 WrapPanel
        var currentWrap = new WrapPanel
        {
            HorizontalAlignment = HorizontalAlignment.Stretch,
            VerticalAlignment = VerticalAlignment.Top,
        };
        groupPanel.Children.Add(currentWrap);
        
        foreach (var item in group.Items)
        {
            if (item is string s && s == "---")
            {
                // 换行：结束当前 WrapPanel，开新的
                currentWrap = new WrapPanel
                {
                    HorizontalAlignment = HorizontalAlignment.Stretch,
                    VerticalAlignment = VerticalAlignment.Top,
                    Margin = new Thickness(0, 4, 0, 0),
                };
                groupPanel.Children.Add(currentWrap);
                continue;
            }
            
            if (item is SensorCardViewModel cardVM)
            {
                // 正常模式：直接渲染卡片
                currentWrap.Children.Add(CreateSensorCard(cardVM));
            }
        }
        
        groupBorder.Child = groupPanel;
        _rowsStackPanel.Children.Add(groupBorder);
    }
}
```

### 6.3 窗口缩放行为

- 每行内部：WrapPanel 决定什么时候折行——窗口够宽一行全展开，窗口窄了就折
- **折下来的卡片仍在该行的灰底圆角框内**——和下一行的卡片不会混淆
- 行与行之间：永远垂直排列，12px 间距。有组名标签则更明显，无标签时灰底框自身就是界线
- 卡片**默认** 160×200。各类型可根据需要覆写（如未来滑杆卡可设 240×200）。同一排内卡片高度不一由 WrapPanel 顶部对齐，组背景框自动适配最高卡。

### 6.4 和扁平列表的区别

| | 之前（扁平列表+换行标记） | 现在（组） |
|------|------|------|
| 加错行 | 删了在正确位置重建 | 侧栏拖卡片跨行移动 |
| 行命名 | 无法挂到换行标记上 | 组自带 name 字段 |
| 跨行插入 | 无此概念 | 侧栏选目标行 → `[+卡]` |
| prefs.json | 一维数组 + 特殊 "---" 标记 | 二维结构，自然表达 |

<a id="78-添加卡片流程"></a>
### 7.8 添加卡片流程

两种入口，弹出同一小窗：

| 入口 | 位置 | 卡片加到哪 |
|------|------|------------|
| 主区虚线 `[+]` | 编辑模式下每两张卡片之间 | 插在该位置，右边卡片自动右移 |
| 侧栏 `[+ 卡片]` | 每行末尾 | 行尾 |

虚线 `[+]` 是编辑模式渲染时位于两个 `CreateSensorCard` 之间的一个悬浮热区，字符 `+` 浅灰色，悬停变色弹出。点它 = 选中了插入位置。

**弹窗定位**——和现有 `ShowColorPickerPopup` 同机制，用 WPF `Popup` + `PlacementTarget`：

```csharp
var popup = new Popup {
    PlacementTarget = sender as UIElement,  // 点的哪个 [+] → 窗贴哪
    Placement = senderInSidebar ? PlacementMode.Left : PlacementMode.Right,
    StaysOpen = false,                      // 点别处自动关
};
```

| 入口 | Placement | 效果 |
|------|-----------|------|
| 主区 `[+]` | `Right` | 小窗贴着 `[+]` 右侧弹出 |
| 侧栏 `[+ 卡片]` | `Left` | 小窗贴着按钮左侧弹出，避免被侧栏右边界截断 |

两种入口弹出同一模板选择小窗：

```
┌─ 添加卡片 ──────────────┐
│                          │
│ 模板：[温度        ▾]    │
│       ─────────          │
│       温度               │
│       湿度               │
│       气压               │
│       状态               │
│       开关               │
│                          │
│ 名字：[______________]   │
│                          │
│        [添加]  [取消]    │
└──────────────────────────┘
```

- 模板：5 类下拉选择
- 名字：用户输入，即卡片显示名 + 协议名
- 点 `[添加]` → 卡片创建。主区 `[+]` 入口 → 卡片插入该位置，右边卡片自动右移（WrapPanel 内顺序由 `Row.Cards` 列表控制）。侧栏 `[+ 卡片]` 入口 → 卡片加到行尾
- 自动创建（固件数据驱动）的卡片走同名逻辑——不弹窗，静默建，同样加到行尾

<a id="710-卡片行内排序-按钮"></a>
### 7.10 卡片行内排序（↑↓ 按钮）

侧栏每张卡片条目旁的 `[↑]` / `[↓]` 控制该卡在当前行内的左右位置。侧栏从上到下 = 该行内从左到右。

| 卡位置 | 显示的按钮 | 说明 |
|:------|:----------|------|
| 行首（第一张） | `[↓]` | 只能往后移 |
| 中间 | `[↑] [↓]` | 两个方向都有 |
| 行尾（最后一张） | `[↑]` | 只能往前移 |
| 行内仅 1 张 | 都不显 | 没有可换的对象 |

点 `[↓]` → 该卡与后面一张交换位置 → `Row.Cards.Move(i, i+1)` → 主区刷新。点 `[↑]` → 与前面一张交换。

实现：侧栏列表每个卡片条目判断 `index == 0` / `index == count-1` 来显隐按钮，点击回调执行 `Row.Cards.Move` + `RefreshAllRows()`。无需新 ViewModel 方法。`RefreshAllRows()` 全量重建确保每次移动后 ↑↓ 按钮自动跟随新位置——从中间移到行尾后 `[↑][↓]` 自动变成 `[↑]`，反之亦然。

<a id="711-卡片改名f2"></a>
### 7.11 卡片改名（F2）

**为什么需要 F2**：滑杆改名只需编辑→侧栏改名字框，没有模板类型层。传感卡片有——`[sensor,pressure,大气压,...]`，"pressure"是建卡时选定的模板类型，改名只改第三字段"大气压"。没有 F2 改个名就得删卡→重选模板→重输名字。F2 跳过删+重选，只改名。

| 操作 | 行为 |
|------|------|
| 编辑模式下选中卡片 → 按 F2 | 标题文字变 TextBox，全选蓝色高亮 |
| 全选态直接打字 | 覆盖全部旧文字 |
| 全选态点一下蓝色区域 | 取消全选，光标到点击位，可在字符间插改 |
| 回车 | 确认，`vm.Name` 更新，卡片标题立即刷新 |
| Esc 或焦点移出 | 取消，恢复原名 |

**改名不影响**：模板类型（竖条色/波形/开关按钮样式）不变。`DeletedNames` 里的旧名不记——那是删卡时记的。

**固件端需同步**：改完名后 `FindByName` 用新名字匹配。固件如果还在发旧名，数据只在接收区滚动，不进卡片。

<a id="712-编辑模式-ui-创建要点"></a>
### 7.12 编辑模式 UI 创建要点

- 编辑模式进入时：`_sensorVM.IsEditMode = true` → `RefreshAllRows()`（重建主区含 × 和 [+]）
- 编辑模式退出时：`_sensorVM.IsEditMode = false` → `RefreshAllRows()`（重建主区恢复正常）
- 和按键/滑杆一致——编辑态下原有交互（开关点击等）禁用，避免冲突

<a id="八引导新-ai-协助添加卡片五问讨论法"></a>
## 八、引导新 AI 协助添加卡片——五问讨论法

如果换了一个 AI 来帮你加新卡片（比如光照卡、PM2.5 卡、滑杆卡），按下面这个固定流程讨论。五问走完蓝图就有了，然后再动手写代码。

### 8.1 五问流程

| 顺序 | 问题 | 举例：如果要加光照卡 |
|:-----|:-----|:-----|
| **① 协议格式** | `[sensor,light,光照,1200,80]`——几个字段？每个什么含义？ | 5 字段：大类 sensor、子类型 light、卡片名、lux 值、百分比 |
| **② 参数规则** | 哪些必发？哪些可选？哪个可能中段留空？ | lux 值必发，百分比可选。传感器坏了 lux 读不到 → `[sensor,light,光照,,80]` |
| **③ 卡片视觉** | 竖条什么色？多宽？有波形吗？有进度条吗？特殊控件？ | 竖条紫色 `#9C27B0`，默认 160×200，有波形（面积图）。无进度条——光照不是百分比概念。无特殊控件 |
| **④ 协议归属** | sensor（被动接收）还是 ctrl（主动发串口）？ | sensor——光照是传感器被动上报 |
| **⑤ 写进文档** | 讨论结果落笔，不靠聊天记录回忆 | 更新设计方案第四章 + 开发计划2 |

### 8.2 用户（Encaron）的典型提问模式

从过往讨论中提炼——你未来加新卡时大概会这样问：

> *"我要加一个滑杆卡，比普通卡长一点，里面有个 Slider 控件可以拖可以点，拖完发串口命令。竖条你用紫色。上面显示当前值，下面一个 Slider。有波形吗？应该没有，因为滑杆是人控的不是传感器读的。"*

AI 收到这种描述后，要做的是：

1. 把"紫色"翻译成具体 hex → `#9C27B0`
2. 把"比普通卡长一点"翻译成宽度覆写 → `240px`（默认 160）
3. 确认"发串口命令" → 协议大类用 `ctrl`
4. 确认"无波形" → 跳过 CreateMiniPlot
5. 给协议格式草案 → `[ctrl,slider,电机转速,2450]`
6. 和你逐条确认五问

### 8.3 用户决策模式

- **颜色、对齐、命名**——特别敏感，但不用设计术语，讲效果不讲原则
- **做决定可能反复**——看到方案后需要时间想，"可以"/"同意"说出来才是定了
- **定了就写进文档**，后续不要推翻重问
- **喜欢类比**——把他的硬件经验（I²C、面包板、跳线）类比软件设计，理解更快
- **前面 6 个面板是 Array 思维**（同类型排列），**传感面板是 Cluster 思维**（异质混搭）——他最满意这个架构。新功能不要掐死这灵活性

### 8.4 代码改动清单（每加一卡）

| 文件 | 加什么 | 行数 |
|:-----|:-----|:-----|
| `SensorCardViewModel.GetAccentHex()` | switch 加一行：新 Type → 色值 | 1 |
| `SensorCardViewModel.GetUnit()` | switch 加一行：新 Type → 单位字符串 | 1 |
| `Panels/Sensors.cs` 的 `CreateSensorCard()` | 加 else if：渲染逻辑（控件组装） | ~30 |
| 协议路由（`OnLineReceived`） | **不动**——sensor/ctrl 大类已扛所有子类型 | 0 |

### 8.5 建卡硬性约束

1. **Type 是 string**，不是 enum。永远小写英文。prefs.json 存字符串，新 Type 不破旧数据。
2. **高度固定 200px**。宽度可覆写。防止同一排高低参差。
3. **左竖条色 = 主值色 = 波形色 = GetAccentHex 返回值**。三处同源。
4. **卡片名唯一**：FindByName 跨所有组搜索。同名 = 同一张卡。改名后旧名失效。
5. **编辑模式下所有交互禁用**。开关/滑杆不可点击。
6. **代码创建的 Popup 必须设 PlacementTarget。**
7. **中段留空用双逗号**——不发明新分隔符或移位逻辑。
8. **VM 不放渲染逻辑**：`GetAccentHex`/`GetUnit` 是仅有的两个传感专属方法，Type/Name/Value/History/序列化全是通用的。渲染函数（`CreateSensorCard`、迷你波形、进度条）留在 `Panels/Sensors.cs`。不在 VM 加 `HasWaveform`/`HasProgressBar` 等虚属性。

## 十一、FixedSizeQueue

```csharp
// Core/Helpers/FixedSizeQueue.cs
public class FixedSizeQueue<T>
{
    private readonly Queue<T> _queue = new();
    private readonly int _capacity;

    public FixedSizeQueue(int capacity) { _capacity = capacity; }

    public void Enqueue(T item)
    {
        _queue.Enqueue(item);
        while (_queue.Count > _capacity) _queue.Dequeue();
    }

    public List<T> GetSnapshot() => _queue.ToList();
    public int Count => _queue.Count;
}
```
