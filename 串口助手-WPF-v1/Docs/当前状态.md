### 架构决策速查
| 决策 | 原因 |
|------|------|
| partial class 拆分而非 Service 类 | 零代码改动，编译产物一致，对新手更安全 |
| `{DynamicResource}` 全量替换（95 处） | StaticResource 只解析一次，换 Resources 不传播 |
| **AvalonEdit 替代 RichTextBox** | 虚拟化渲染（仅 ~35 可见行有 UI 元素），拖窗口丝滑，代码量减半 |
| AvalonEdit 原生行号（`ShowLineNumbers="True"`） | 省掉自建 ItemsControl 的 ~100 行滚动同步/动态列宽/显隐逻辑 |
| `LogColorizer`（DocumentColorizingTransformer）替代 `Paragraph.Tag` | 主题切换时 `TextView.Redraw()` 重新着色，按 `_lineRoles` 列表查询每行角色 |
| `_lineRoles: List<string>` 追踪每行角色 | 行于行一一对应，裁剪/清空时同步维护 |
| `DataReceived -=` 在 `CloseSerialPort` 前退订 | 防死锁：`serialPort.Close()` 不再等待卡在 `Dispatcher.Invoke` 的回调 |
| `DataReceived +=` 在 `OpenSerialPort` 重新订阅 | `-=` 后如果不 `+=`，重开串口将不再接收数据 |
| `Canvas.Left` 做滑动开关而非 TranslateTransform | TranslateTransform 在 ControlTemplate Trigger 中不可达 |

| # | 规则 | 说明 |

## 项目背景

### 起源
- 原始项目：**B 站 UP 主江科大**编写的 **C# WinForms 串口助手 V1.1**（位于 `串口助手/`，不动）
- 江科大的串口逻辑部分完成度极高，是本项目的基石
- 用户做了两轮修改：默认参数调整（115200/文本/UTF-8）+ WinForms 界面美化

### 为什么又开了 WPF 版
- WinForms 天花板太低（无圆角、无动画、无原生阴影）
- Web 前端不可行（浏览器不能直接访问串口），WPF 是最佳折中——串口逻辑 100% 复用、UI 脱胎换骨
- `串口助手-WPF/` 与 WinForms 原版 **并存、互不影响**

### 目标框架
- **.NET Framework 4.8**，VS2022 编译，双击 .exe 即用

## 已知注意事项

1. **等宽字体**：首选 `Sarasa Mono SC`，回退 `Consolas` → `Courier New`
2. **HEX 模式发送回显**：超过 80 字符截断预览
3. **日志行数上限**：`MaxLogLines = 2000`，超出时一次裁剪 `CropMargin = 500` 行；阈值已修正为 `> MaxLogLines`（之前为 `MaxLogLines + CropMargin/2`）
4. **行缓冲定时器**：100ms 空闲后强制输出残留文本
5. **窗口设置存储**：`%LocalAppData%\串口助手WPF\window.cfg`，加载分 `LoadWindowSettings()` + `LoadPreferences()` 两阶段
6. **快捷发送存储**：`%LocalAppData%\串口助手WPF\quick_sends.cfg`
7. **崩溃日志**：`%LocalAppData%\串口助手WPF\crash.log`
8. **ComboBox 模板**：已含 `IsEditable` Trigger（`True`→隐藏 ContentSite；`False`→折叠 PART_EditableTextBox），新增 ComboBox 如设为可编辑需确认模板适配
9. **AvalonEdit 依赖**：`ICSharpCode.AvalonEdit.dll` 必须与 exe 同一目录；已在 .csproj 中通过 `<HintPath>` 引用 `Lib\` 目录
10. **行号**：AvalonEdit 原生行号（`ShowLineNumbers`），颜色通过 `LineNumbersForeground="{DynamicResource TextMutedBrush}"` 跟随主题
11. **主题切换着色**：`LogColorizer`（`DocumentColorizingTransformer`）按 `_lineRoles` 列表查询每行角色；`ApplyTheme()` 调用 `editor.TextArea.TextView.Redraw()` 触发重着色
12. **角色追踪**：`_lineRoles: List<string>` 与 editor Document 行一一对应；追加时 `_lineRoles.Add(role)`，裁剪时 `_lineRoles.RemoveAt(0)`，清空时 `_lineRoles.Clear()`
13. **DataReceived 生命周期**：构造函数首次订阅；`CloseSerialPort` 退订（防死锁）；`OpenSerialPort` 重新订阅（`-=` + `+=` 确保恰好一个 handler）
14. **搜索**：`SearchPanel.Install(editor.TextArea)` 于 `InitEditor()` 中启用，Ctrl+F 弹出搜索栏，位于 `ICSharpCode.AvalonEdit.Search`
15. **Enter=发送**：使用 `PreviewKeyDown`（隧道事件），不可改回 `KeyDown`
16. **C# 常量**：`PrimaryColor`/`LogSystemColor`/`LogSentColor`/`LogReceivedColor`/`StatusDotIdle` 在 `ApplyTheme()` 中随主题切换
17. **{DynamicResource}**：全 XAML 96 处（含新增 LineNumbersForeground）；新增引用也必须用 Dynamic
18. **DTR/RTS**：连接中可随时切换
19. **流控**：打开串口后禁用（不可动态切换）
20. **暂停缓冲**：`_pausedLines` 上限 2000，满时写系统消息提醒
21. **partial class 拆分**：6 个文件共属 `MainWindow` 类；新加 .cs 必须同步更新 .csproj
22. **版本号**：状态栏右下角显示 `v1.0.0`，`AssemblyName` 已改为 `Serial Monitor`
23. **软件名称**：窗口标题 `Serial Monitor`，崩溃弹窗标题 `Serial Monitor — Error`

## 本轮关键决策速查（供新 AI 快速理解"为什么"）

| 决策 | 原因 |
|------|------|
| AvalonEdit 替代 RichTextBox | 虚拟化渲染：仅 ~35 个可见行有 UI 元素；拖窗口/滚动`O(1)`而非`O(n)` |
| 去掉批量队列（`_logBatch`/`_batchFlushTimer`/`FlushLogBatch`） | AvalonEdit 追加一行 ≈ 字符串操作，不再需要缓冲合并 |
| `LogColorizer`（`DocumentColorizingTransformer`）替代 `Paragraph.Tag` | 无 Tag 机制时用独立 `_lineRoles` 列表追踪每行角色；主题切换时 `Redraw()` 触发重新着色 |
| `_lineRoles: List<string>` 与 editor 行同步 | `Add` 追加 / `RemoveAt(0)` 裁剪 / `Clear` 清空 |
| `DataReceived -=` 在 Close 前 | 防 `serialPort.Close()` 等待卡在 `Dispatcher.Invoke` 的回调 → 死锁 |
| `DataReceived +=` 在 Open 后 | 退订后必须重新订阅，否则重开串口不接收 |
| ComboBox 模板加 `IsEditable` Trigger | `ContentPresenter` 和 `PART_EditableTextBox` 重叠渲染导致下拉重影 |
| 崩溃日志写入 `crash.log` | `AppDomain.UnhandledException` + `Dispatcher.UnhandledException` |
| 暂停缓冲满时写系统消息 | 消除静默数据丢失 |
| `CropMargin`/`SmartScrollLockPixels` 等常量 | 消除魔法数字；`EmergencyFlushThreshold`/`BatchFlushIntervalMs` 已随 AvalonEdit 迁移清理 |
| `SearchPanel.Install(editor.TextArea)` | AvalonEdit 内置搜索，一行启用，替代 RichTextBox 时代有 bug 的自研搜索 |
| `LineNumbersForeground="{DynamicResource TextMutedBrush}"` | 行号颜色跟随亮/暗主题自动切换，消除最后一个视觉不一致点 |
| `SendRaw(content, appendLineEnding)` | 快捷发送按钮已有内置换行，不能重复追加 |
| `LoadPreferences()` 独立于 `LoadWindowSettings()` | ComboBox 必须先填充 Items 才能设置 SelectedIndex |
| C# 7.3 | .NET Framework 4.8 默认值 |
| `{DynamicResource}` 全局（95 处） | Static 不传播；Dynamic 追踪变化 |
| partial class 而非 Service 类 | 零代码改写、新手友好 |
