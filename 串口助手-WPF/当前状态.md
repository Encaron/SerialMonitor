### 架构决策速查
| 决策 | 原因 |
|------|------|
| partial class 拆分而非 Service 类 | 零代码改动，编译产物一致，对新手更安全 |
| `{DynamicResource}` 全量替换（95 处） | StaticResource 只解析一次，换 Resources 不传播 |
| 行号用独立 ItemsControl 而非内嵌 Run | 导出/复制不含行号；列宽可独立扩容 |
| `Canvas.Left` 做滑动开关而非 TranslateTransform | TranslateTransform 在 ControlTemplate Trigger 中不可达 |
| `FindVisualChild<ScrollViewer>` hook 滚动事件 | RichTextBox 内部 ScrollViewer 不公开暴露 |
| `Paragraph.Tag` 存角色标签 | 主题切换时按角色重新着色已有文字 |

## 项目背景

### 起源
- 原始项目：**B 站 UP 主江科大**编写的 **C# WinForms 串口助手 V1.1**（位于 `串口助手/`，不动）
- 江科大的串口逻辑部分完成度极高，是本项目的基石
- 用户做了两轮修改：默认参数调整（115200/文本/UTF-8）+ WinForms 界面美化

### 为什么又开了 WPF 版
- WinForms 天花板太低（无圆角、无动画、无原生阴影）
- Web 前端不可行（浏览器不能直接访问串口），WPF 是最佳折中——串口逻辑 100% 复用、UI 脱胎换骨
- `串口助手-WPF/` 与 WinForms 原版 **并存、互不影响**

### 目标框架
- **.NET Framework 4.8**，VS2022 编译，双击 .exe 即用

## 新增功能详解

> **第一轮**（`0cb32fd` ~ `80de59a`）：流量统计、快捷发送、HEX格式化、行号显示。  
> **第二轮**（`7fbb35c` ~ `83fbdb2`）：换行符、发送后清空、自动重连、流量持久化、系统消息独立、滑动开关。

### 流量统计（第一轮）
- **位置**：状态栏，圆点和状态文字右侧
- **格式**：`TX: 1.2KB ↑  RX: 8.5KB ↓`（自动换算 B/KB/MB）
- **串口关闭时**：`TX: -- ↑  RX: -- ↓`（灰色）
- **右键**：点击流量文字 → 重置计数归零 + 日志提示
- **统计时机**：`SendData()` 累加 TX、`serialPort_DataReceived` 累加 RX

### 快捷发送按钮面板（第一轮）
- **位置**：发送区卡片内，标题下方、输入区上方（WrapPanel 自动换行）
- **预置指令**：`AT` / `AT+CWLAP` / `AT+CWMODE=1` / `AT+CIFSR` / `AT+RST`
- **交互**：左键 → 立即发送（调用 `SendRaw()`）；右键 → 编辑标签 / 删除
- **添加**："＋" 按钮 → 将当前输入区内容保存为快捷发送（弹出输入对话框命名）
- **持久化**：`%LocalAppData%\串口助手WPF\quick_sends.cfg`（tab 分隔格式）
- **架构**：提取 `SendRaw(string content)` 方法，快捷发送和 `SendData()` 共用

### HEX 输入实时格式化（第一轮）
- **触发**：发送模式为 HEX 时，`tbSend_TextChanged` 自动格式化
- **规则**：过滤非法字符（只保留 `[0-9A-Fa-f]`）→ 转大写 → 每两字符插空格
- **光标保持**：统计 hex 字符数映射到格式化后位置，防跳动
- **防重入**：`_isHexFormatting` 标志
- **模式切换**：切换到 HEX 时自动格式化现有内容

### 行号显示（第三轮重构：独立面板）
- **方案**：VS Code 风格——左侧独立 `ScrollViewer` + `ItemsControl`（`icLineNumbers`），与 RichTextBox 分两列
- **滚动同步**：`rtReceive_Loaded` → `FindVisualChild<ScrollViewer>` 找到内部滚动条 → `RtScrollViewer_ScrollChanged` 同步 `svLineNumbers.ScrollToVerticalOffset`
- **列宽动态扩容**：4位48px起步 → 10000行扩至52px → 100000行扩至58px → 1000000行扩至64px（封顶）
- **即时显隐**：`svLineNumbers.MinWidth="48"` 保证空面板可见，CheckBox 切换 `Visibility` 即时生效
- **自适应**：Grid 列 `Auto` + `*`，取消行号后 RichTextBox 自动撑满
- **数据一致性**：追加/清空/裁剪均同步操作 `icLineNumbers.Items` 和 `rtReceive.Document.Blocks`
- **导出无污染**：行号不在 RichTextBox 文本流中，导出/复制不含行号
- **主题切换**：行号颜色来自 `{DynamicResource TextMutedBrush}`，随主题自动切换

### ❌ 搜索功能（已回退）
- **原因**：WPF RichTextBox 的 `GetPositionAtOffset` 在 Paragraph 层级计数包含元素边界标记而非纯字符，导致光标位置错乱和崩溃
- **结论**：在该架构下难以根治，放弃

### 换行符可选（第二轮）
- **位置**：发送设置卡片 → ComboBox
- **选项**：`\r\n`（默认）、`\n`、`\r`、`无`
- **文本模式**：追加换行字符串到内容末尾（如 `AT` → `AT\r\n`）
- **HEX 模式**：追加 HEX 字节（如 `41 54` → `41 54 0D 0A`）
- **架构**：`SendRaw(content, appendLineEnding: true)` 仅主发送区触发，快捷发送按钮不受影响
- **辅助方法**：`GetLineEndingText()` / `GetLineEndingHex()` 提供映射

### 发送后自动清空发送区（第二轮）
- **位置**：发送设置卡片 → `☑ 发送后清空发送区` CheckBox
- **行为**：`SendData()` 调用 `SendRaw()` 之后执行 `tbSend.Clear()`
- 仅主发送区触发，快捷发送按钮不受影响

### 串口热插拔自动重连（第二轮）
- **检测**：`WM_DEVICECHANGE` → `DBT_DEVICEARRIVAL`（设备重新插入）
- **机制**：`lastPortName` 记住断开前端口 → `reconnectTimer` 每 500ms 轮询 → 最多 10 次（5 秒）超时
- **日志**：重连成功/超时均有系统日志提示
- **开关**：`☑ 自动重连` CheckBox，默认关

### 流量计数持久化（第二轮）
- **行为**：`OpenSerialPort()` 检测 `chkPersistTraffic.IsChecked`，勾选时跳过 `txByteCount/rxByteCount = 0`
- **开关**：`☑ 串口打开时不重置流量计数` CheckBox，默认关

### 系统消息独立显示（第二轮）
- **位置**：接收区卡片内，RichTextBox 上方（DockPanel.Top）
- **控件**：ListBox（2行高 36px，MinHeight=18、MaxHeight=72，超出自动滚动）
- **配色**：`SecondaryHoverBgBrush`底 + 左侧3px `InputBorderBrush`边框，和数据区视觉区分
- **上限**：最多 50 条，超出删除旧条目；自动滚到底
- **开关**：接收设置 `☑ 系统消息独立显示` CheckBox，默认勾选
- **取消后**：`LogSystem` 回流 RichTextBox → 恢复旧行为
- **动画**：勾选/取消时 200ms Opacity 淡入淡出
- **导出**：导出日志不含系统消息（只导 RichTextBox）

### CheckBox → ToggleSwitch 滑动开关（第二轮）
- **模板**：Canvas + 40×22 圆角滑轨（`InputBorderBrush`/`PrimaryBrush`）+ 18×18 白圆钮
- **动画**：`AnimateToggleSwitch()` → `Canvas.Left` 2↔20 滑动（200ms）
- **触发**：`IsChecked=True` Trigger 同步圆钮位置（`Canvas.Left=20`），解决初始位置错误
- **覆盖**：隐式 CheckBox Style，所有 7 个 CheckBox 自动获得滑动开关外观
- **暗色**：滑轨/圆钮色值均来自 `{StaticResource}`，亮暗自动适配

### 为什么用 partial class 而非 Service 类
- 零代码改写：只是物理剪切粘贴，不改任何一行逻辑
- 编译产物一致：IL 和原来完全一样
- 风险最低：所有字段/方法/控件引用天然互相可见
- 新手友好：打开哪个文件就看哪个功能块，不用理解依赖注入

### 本轮关键决策速查（供新 AI 快速理解"为什么"）

| 决策 | 原因 |
|------|------|
| `SendRaw(content, appendLineEnding)` 而非直接在 SendRaw 内追加 | 快捷发送按钮已有内置换行，不能重复追加 |
| 系统消息用 DockPanel+ListBox 而非第二个 RichTextBox | 轻量、样式可控、不用再搞一套 FlowDocument |
| CheckBox 模板用 Canvas+Canvas.Left 而非 TranslateTransform | TranslateTransform 是 Freezable，ControlTemplate Trigger 找不到它 |
| `LoadPreferences()` 独立于 `LoadWindowSettings()` | ComboBox 必须先填充 Items 才能设置 SelectedIndex |
| `AnimateToggleSwitch()` 在每个 CheckBox 事件处理器中调用 | WPF 的 EventTrigger 不支持 Checked/Unchecked，只能 code-behind |
| `window.cfg` 用 `File.WriteAllLines` 覆盖写 | 文件不增长，14 行永远 <400 字节 |
| C# 语言版本锁定 7.3 | .NET Framework 4.8 的默认值，switch 表达式等新语法不能用 |
| `{StaticResource}` → `{DynamicResource}` 全局替换（95 处） | `{StaticResource}` 只在 XAML 加载时解析一次，`ApplyTheme()` 换 Resources 不传播；`{DynamicResource}` 追踪变化 |
| `Paragraph.Tag` 存角色标签（system/sent/received） | 主题切换时 `RefreshRichTextBoxColors()` 遍历所有 Paragraph，按角色重新着色 |
| 行号用独立 ItemsControl 而非内嵌 Run | 导出/复制不含行号；可独立显隐；列宽动态扩容不污染文本流 |
| `FindVisualChild<ScrollViewer>` hook 滚动事件 | RichTextBox 内部 ScrollViewer 不公开暴露，需通过视觉树遍历获取 |
| `PrimaryBrush` 加入 ThemeMap（暗色 `#0E639C`） | 旧版亮暗同色，暗色下蓝按钮过于刺眼 |
| partial class 拆分而非 Service 类 | 零代码改写、编译一致、风险最低、新手友好 |
| 暂停用独立 `_pausedLines` 列表 + `AppendColoredLineUnbuffered` | 恢复时一次性回放，不清空缓冲可能导致内存泄漏 |
