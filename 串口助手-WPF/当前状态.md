## 项目背景

### 起源
- 原始项目：江科大 UP 主编写的 **C# WinForms 串口助手 V1.1**（位于 `串口助手/`）
- 用户 Fork 后做了两轮修改：
  1. **默认参数调整**：波特率 9600→115200、收发模式 HEX→文本、编码 GBK→UTF-8
  2. **WinForms 界面美化**：微软雅黑全局、更纱黑体等宽文本框、GroupBox 白底卡片、按钮扁平配色

### 为什么又开了 WPF 版
- WinForms 再做下去天花板太低（无圆角、无动画、无原生阴影、GroupBox 蚀刻边框年代感强）
- 用户下载了前端设计 skills（`design-taste-frontend` 等），想要更现代的 UI
- Web 前端不可行（浏览器不能直接访问串口），WPF 是最佳折中——串口逻辑 100% 复用、UI 脱胎换骨
- 于是新建 `串口助手-WPF/`，与 WinForms 原版**并存、互不影响**

### 目标框架
- **.NET Framework 4.8**（与 WinForms 版一致，用户系统自带运行时）
- VS2022 编译，双击 .exe 即用

## 当前功能清单

### ✅ 已完成

#### 串口核心（从 WinForms Form1.cs 直接移植，逻辑零改动）
- 串口参数配置：串口号（动态扫描）、波特率（默认 115200）、数据位（8）、停止位（1）、校验位（无）
- 打开/关闭串口、USB 热插拔检测（WM_DEVICECHANGE）
- 接收模式：HEX / 文本模式（默认文本）
- 发送模式：HEX / 文本模式（默认文本）
- 编码：GBK / UTF-8（默认 UTF-8）
- 4 个数据转换方法：`BytesToHex` / `HexToBytes` / `BytesToText` / `TextToBytes`（**切勿改动**）
- 接收字节缓冲区：跨 DataReceived 拼接多字节字符（GBK/UTF-8）

#### UI 特性
- **布局**：左收发右设置，Grid 弹性分栏
- **卡片**：白底圆角 `CornerRadius=6` + 细边框 `#E0E0E0`
- **按钮**：主按钮（蓝色 `#0078D4`）/ 次要按钮（灰色边框）/ 连接状态（绿色 `#10B981`）
- **ComboBox**：扁平样式，统一 28px 高
- **状态栏**：绿色/灰色圆点 + 连接状态文字 + 端口信息
- **图标**：复用原版 `icon.ico`

#### 接收区（RichTextBox 彩色日志）
- **系统消息**（灰色 `#999`）：打开/关闭串口、时间戳开关、回显开关
- **发送回显**（蓝色 `#0078D4`）：已发送内容 + 编码信息，可通过 CheckBox 关闭
- **接收数据**（深色 `#2D2D2D`）：串口实际收到的数据
- **时间戳**：格式 `HH:mm:ss:fff`，可通过 CheckBox 开关
- **行缓冲**：DataReceived 碎片自动拼接成完整行再输出，100ms 无数据强制输出

#### 发送区
- Enter = 发送，Shift+Enter = 换行
- `MinHeight=68`（约 3 行）→ `MaxHeight=140`（约 7 行，超出滚动）
- 底部同排：左灰色提示 `Enter 发送 · Shift+Enter 换行` + 右按钮（清空/发送）

#### 字体
- UI：微软雅黑 9pt（窗口级继承）
- 标题：13pt SemiBold
- 数据区：等宽 `Sarasa Mono SC, Consolas, Courier New` **12pt**

## 编译 & 运行

### 编译
在 VS2022 中打开 `串口助手-WPF.csproj`，切换 Release 配置，生成解决方案。

或命令行：
```powershell
& "D:\Program Files\Microsoft Visual Studio\2022\Community\MSBuild\Current\Bin\MSBuild.exe" `
  "串口助手-WPF.csproj" /t:Build /p:Configuration=Release
```

输出：`bin\Release\串口助手 WPF.exe`

### 运行
直接双击 `串口助手-WPF/串口助手 WPF.exe`。需要 Windows 系统已安装 .NET Framework 4.8（Win10+ 自带）。

## 已知注意事项

1. **等宽字体回退**：首选 `Sarasa Mono SC`（更纱黑体），未安装时自动回退 `Consolas` → `Courier New`
2. **HEX 模式发送回显**：超过 80 字符会截断预览，避免日志刷屏
3. **日志行数上限**：`MaxLogLines = 2000`，超出自动裁剪旧行
4. **行缓冲定时器**：100ms 空闲后强制输出残留文本，关闭串口/切换模式也会触发
5. **更纱黑体 Mono SC**：推荐从 [GitHub Releases](https://github.com/be5invis/Sarasa-Gothic/releases) 下载安装
