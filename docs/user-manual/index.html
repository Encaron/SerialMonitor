工程细节

不是"能跑就行"——每个交互都认真想过。

🫀

死人不报丧

状态卡每 500ms 发一次心跳。PC 端 2 秒没收到 → 自动绿变红、online → offline。固件挂了不需要发离线消息——"死人不会打电话报丧"。

🎛

Cluster 思维

8 类卡片，每种长得不一样。温度有波形、湿度有进度条、气压 PC 端自算百分比、开关是 iOS 胶囊滑块。同排混搭，WrapPanel 自然折行。

✏

编辑模式

卡片间虚线 [+] 悬停弹性展开 8→16px 不挤卡片。F2 改名跳过删卡重选。删组二次确认 3 秒倒计时——不靠弹窗打断。

🔄

F5 增量同步

PC 画板拖图形时每次只发 1 帧——[draw,set,id,...]。MCU 端本地维护图形数组，收到后本地全量重绘。不把整张画布几十条命令重发一遍。

🖱

调度法

拖滑杆时 InvalidatePlot 走 DispatcherPriority.Background——WPF 优先处理 MouseMove，空闲才渲染波形。拖拽永远流畅，松手瞬间恢复 30Hz。

📋

文档不写"略"

72 条测试 29ms 全绿。CubeMX 重新生成后 6 个必补坑全写清——栈 1K→4K、-u _printf_float、TIM6_DAC_IRQHandler 永远不生成……嵌入式开发者一看就知道踩过坑。

📉

迷你波形不糊不单薄

1px 轮廓线保持峰谷细节 + 30% 透明填充给视觉重量。加粗线条 2px+ 会在 140px 宽画布上抹平相邻点差异——面积图方案不碰线宽。

🪫

优雅降级

传感器读回异常？中段留空双逗号 [sensor,temp,温度,,45] → 主值显示 -- 不进波形，辅助参数照常显示。不报错、不丢数据、不污染图表。

为开发者设计

VS Code 风格三栏布局、暗色主题、扁平化设计。嵌入式开发者一眼就能上手。

🌙 VS Code 暗色主题

仿 VS Code Dark+ 配色，21 色 DynamicResource 原地变色不闪烁。从 VS Code 切过来零学习成本——深色背景、蓝色强调、绿色状态灯，肌肉记忆完全复用。

🌐 国际化，国际友人能用

~550 条翻译全覆盖，中英一键瞬切。ComboBox、系统日志、协议预览、错误提示——全线英文覆盖。GitHub README 双语，STM32 C 库文档双语。

📐

三栏布局，VS Code 同款交互

左侧图标栏（Activity Bar）→ 侧面板 220px → 主内容区。标签页点图标切换、侧栏内容跟随图标走、选中态左竖线指示器——和 VS Code 一模一样的操作逻辑。

⚡

一处注册，自动跟

加新面板只需一行 RegisterTab——图标动效、标签切换、编辑态退出、弹窗主题全部自动处理。复制反馈、Popup 定位、透明按钮模板——框架统一兜底，忘写也不会出 bug。72 条测试 29ms 全绿。
