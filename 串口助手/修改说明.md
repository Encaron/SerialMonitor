# 串口助手 V1.1 — 默认参数修改说明

## 背景

- 此项目为江科大 UP 主编写的串口助手（C# WinForms，原本使用 Visual Studio 2015 创建）
- 目标框架：.NET Framework 4.0（源码兼容 .NET Framework 4.8）
- 功能完善，仅需修改软件启动时的默认参数以匹配 STM32 单片机的常用配置
- **修改原则：最小化改动，不改动源代码结构，保持原有代码风格**
- **最终目标：修改完成后，用户双击根目录下的 `串口助手 V1.1.exe` 直接可用，默认即为 115200 波特率 + 文本模式 + UTF-8 编码**

---

## 修改目标

| 参数 | 当前默认值 | 修改后默认值 |
|------|-----------|-------------|
| 波特率 | 9600 | **115200** |
| 接收模式 | HEX模式 | **文本模式** |
| 接收编码 | GBK | **UTF-8** |
| 发送模式 | HEX模式 | **文本模式** |
| 发送编码 | GBK | **UTF-8** |

---

## 完整操作流程

### 步骤一：用 VS2022 打开工程

1. 打开 **Visual Studio 2022**
2. 菜单栏 → **文件** → **打开** → **项目/解决方案**
3. 选择文件：`串口助手-工程源码\串口助手.sln`
4. VS2022 会弹出"**检查项目是否可升级**"的对话框，点击 **确定**，将项目从 .NET Framework 4.0 升级到 4.8
   - 这一步很正常，代码完全兼容，不影响功能
   - 注：如果 VS2022 没有弹框而直接打开了工程，也属于正常情况，继续下一步即可

> **补充说明（仅当 VS2022 提示缺少 .NET Framework 4.0 目标包时）**：如果 VS2022 提示无法加载项目、缺少 .NET Framework 4.0，则需先安装目标包。具体操作：关闭 VS2022，运行 Visual Studio Installer → 找到 VS2022 → "修改" → "单个组件" → 勾选 ".NET Framework 4.0 目标包" → "修改"按钮安装。安装后再重新打开工程。

### 步骤二：修改源代码

修改的文件在解决方案资源管理器中的路径：`串口助手` → `Form1.cs`

在 `Form1.cs` 的 `Form1_Load` 方法中（约第 203~219 行），修改以下 5 行代码中的数字：

```csharp

// 修改后
cbBaudRate.SelectedIndex = 3;
```

> `cbReceiveMode` 下拉列表项：`"HEX模式"`(0)、`"文本模式"`(1)

#### 3. 接收编码默认值（原第 210 行）


> `cbReceiveCoding` 下拉列表项：`"GBK"`(0)、`"UTF-8"`(1)

#### 4. 发送模式默认值（原第 211 行）

#### 修改后 Form1_Load 完整代码（供核对）

```csharp
private void Form1_Load(object sender, EventArgs e)        //窗口加载事件
{
    cbBaudRate.SelectedIndex = 3;       //控件状态初始化
    cbDataBits.SelectedIndex = 3;
    cbStopBits.SelectedIndex = 0;
    cbParity.SelectedIndex = 0;
    cbReceiveMode.SelectedIndex = 1;
    cbReceiveCoding.SelectedIndex = 1;
    cbSendMode.SelectedIndex = 1;
    cbSendCoding.SelectedIndex = 1;
    btnSend.Enabled = false;
    cbPortName.Enabled = true;
    cbBaudRate.Enabled = true;
    cbDataBits.Enabled = true;
    cbStopBits.Enabled = true;
    cbParity.Enabled = true;
}
```

### 步骤三：编译生成新的 .exe

1. 在 VS2022 顶部工具栏，将编译配置从 **Debug** 切换为 **Release**
2. 菜单栏 → **生成** → **生成解决方案**（或按快捷键 `Ctrl+Shift+B`）
3. 等待底部输出窗口显示"**生成: 1 成功**"，表示编译完成

### 步骤四：替换根目录的 .exe

编译成功后，新的 .exe 文件位于：

```
串口助手-工程源码\串口助手\bin\Release\串口助手 V1.1.exe
```

将上述文件 **复制并替换** 根目录下的旧文件：

```
串口助手\串口助手 V1.1.exe
```

### 步骤五：验证

双击根目录下的 `串口助手 V1.1.exe`，确认：
- 波特率默认显示 **115200**
- 接收模式默认显示 **文本模式**，编码默认显示 **UTF-8**
- 发送模式默认显示 **文本模式**，编码默认显示 **UTF-8**

## 修改范围总结

- **仅修改 1 个文件**：`串口助手-工程源码/串口助手/Form1.cs`
- **仅修改 5 行代码**：`Form1_Load` 方法中的 5 个 `SelectedIndex` 数值
- `Form1.Designer.cs` — **不动**（下拉列表项不变，仅通过索引选择默认值）
- 其他所有文件 — **不动**

---

## 注意事项

- `SelectedIndexChanged` 事件由 `SelectedIndex` 赋值自动触发：当接收/发送模式设为 `1`（文本模式）时，对应的编码下拉框自动变为可用状态（`Enabled = true`），反之 HEX 模式时编码框保持灰色。无需额外处理。
- 修改不影响串口通信逻辑、HEX 收发功能等任何其他功能
- 如果 VS2022 提示需要安装 .NET Framework 4.8 目标包，请按照弹窗提示操作（通常 VS2022 默认自带，无需额外安装）
